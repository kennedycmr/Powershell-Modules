Cmdlet: Set-NcDashboardAlarmThreshold


NAME
    Set-NcDashboardAlarmThreshold
    
SYNOPSIS
    Modify alarm threshold settings for the given metric group.
    
SYNTAX
    Set-NcDashboardAlarmThreshold [-DashboardMetricType] <String> 
    [-PollingInterval <Int32>] [-IsUserNotificationEnabled [<Boolean>]] 
    [-Controller <NcController[]>] [-ZapiRetryCount <Int32>] 
    [<CommonParameters>]
    
    Set-NcDashboardAlarmThreshold [-DashboardMetricType] <String> 
    [-WarningThreshold <Int64>] [-CriticalThreshold <Int64>] -ThresholdUnits 
    <String> [-PollingInterval <Int32>] [-IsUserNotificationEnabled 
    [<Boolean>]] [-Controller <NcController[]>] [-ZapiRetryCount <Int32>] 
    [<CommonParameters>]
    
    
DESCRIPTION
    Modify alarm threshold settings for the given metric group.
    

PARAMETERS
    -DashboardMetricType <String>
        The type of metric being monitored. Possible values:
        "cpu_busy"       - CPU Utilization
        "port_util"      - Port Utilization
        "op_latency"     - Average Client Latency of NFS and CIFS operations
        "aggregate_used" - Storage Aggregate Utilization
        "port_problems"  - Packet Error Ratio
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -PollingInterval <Int32>
        The refresh interval for the alarm dashboard metrics (in seconds).
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -IsUserNotificationEnabled [<Boolean>]
        Specifies whether user notifications are enabled.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Controller <NcController[]>
        A clustered Data ONTAP controller to receive this cmdlet, as embodied 
        by an NcController object.  This parameter is returned by the 
        Connect-NcController cmdlet.  If not specified, the value in the global 
        variable CurrentNcController is used.  In the latter case, if 
        CurrentNcController contains a collection of NcController objects, this 
        cmdlet is invoked on each controller in succession.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -ZapiRetryCount <Int32>
        
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -WarningThreshold <Int64>
        The threshold value that must be reached to generate a warning alarm 
        for this type of metric.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -CriticalThreshold <Int64>
        The threshold value that must be reached to generate a critical alarm 
        for this type of metric.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -ThresholdUnits <String>
        Threshold Units Possible values:
        "none"      - none
        "ms"        - milliseconds
        "percent"   - percent
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    DataONTAP.C.Types.Dashboard.DashboardAlarmThresholdInfo
        
        
    
     
    
NOTES
    
    
        Category: dashboard
        API: dashboard-alarm-threshold-modify
        Family: cluster
    
    --------------  Example 1 --------------
    
    C:\PS>Set-NcDashboardAlarmThreshold -DashboardMetricType port_problems 
    -WarningThreshold 2 -ThresholdUnits percent
    
    
    Set the warning threshold for the dashboard metric type port_problems to 2%.
    
    
    
    DashboardMetricType         PollingIntervalTS   WarningThreshold  
    CriticalThreshold  IsUserNotificationEnabled
    -------------------         -----------------   ----------------  
    -----------------  -------------------------
    port_problems                        00:05:00                 2%            
         5%            False
