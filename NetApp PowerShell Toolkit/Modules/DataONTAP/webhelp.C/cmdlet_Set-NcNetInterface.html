Cmdlet: Set-NcNetInterface


NAME
    Set-NcNetInterface
    
SYNOPSIS
    Modify the attributes of a network interface (LIF).
    
SYNTAX
    Set-NcNetInterface [-Name] <String> [-Vserver] <String> [-Node <String>] 
    [-Port <String>] [-Address <String>] [-Netmask <String>] [-Ipv4LinkLocal 
    [<Boolean>]] [-RoutingGroup <String>] [-DnsDomain <String>] 
    [-FirewallPolicy <String>] [-FailoverPolicy <String>] [-FailoverGroup 
    <String>] [-UseFailoverGroup <String>] [-AutoRevert [<Boolean>]] 
    [-AdministrativeStatus <String>] [-Comment <String>] [-NetmaskLength 
    <Int32>] [-ListenForDnsQuery [<Boolean>]] [-Subnet <String>] 
    [-ForceSubnetAssociation] [-Wwpn <String>] [-DnsUpdateEnabled [<Boolean>]] 
    [-Controller <NcController[]>] [-ZapiRetryCount <Int32>] 
    [<CommonParameters>]
    
    
DESCRIPTION
    Modify the attributes of a network interface (LIF).
    

PARAMETERS
    -Name <String>
        Specifies the logical interface (LIF) name.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Vserver <String>
        Specifies the Vserver name.
        
        Required?                    true
        Position?                    2
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Node <String>
        Specifies the LIF's home node.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Port <String>
        Specifies the LIF's home port.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Address <String>
        Specifies the LIF's IP address.  This element is valid for all data 
        protocols except FCP.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Netmask <String>
        Specifies the LIF's netmask.  This element is valid for all data 
        protocols except FCP.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Ipv4LinkLocal [<Boolean>]
        If true, automatically configure an interface with an IPv4 address 
        (cluster LIFs only).  User can configure an interface by explicitly 
        specifying Address and Netmask or by setting Ipv4LinkLocal to true.
        
        Note: This parameter is not available on Data ONTAP 8.4 and later.
        
        Required?                    false
        Position?                    named
        Default value                false
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -RoutingGroup <String>
        Specifies the routing group, which enables multiple logical interfaces 
        to share a set of routing table entries. For example: 
        
        d192.168.1.0/24 ('d' stands for data LIF and 192.168.1.0/24 is subnet)
        c192.168.1.0/24 ('c' stands for cluster LIF and 192.168.1.0/24 is 
        subnet)
        n192.168.1.0/24 ('n' stands for node management LIF and 192.168.1.0/24 
        is subnet)
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -DnsDomain <String>
        Specifies the unique, fully qualified domain name of the DNS zone of 
        this LIF.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -FirewallPolicy <String>
        Specifies the firewall policy for the LIF.  Default firewall policy is 
        set as per the following table:
        
        LIF Role     Protocols   Default policy
        -------      ---------   --------------
        data            none          mgmt
        data            any           data
        node_mgmt       any           mgmt
        cluster_mgmt    any           mgmt
        cluster         any          cluster
        intercluster    any       intercluster
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -FailoverPolicy <String>
        Specifies the failover policy for the LIF.  For FCP and iSCSI LIFs, the 
        only failover policy is 'disabled'; for NFS, CIFs and fcache LIFs, the 
        default policy is 'nextavail'.  Possible values:
        
        "nextavail" - Next failover target selected based on next available 
        port ID
        "priority"  - Next failover target selected based on lowest available 
        port ID
        "disabled"  - Failover disabled
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -FailoverGroup <String>
        Specifies the failover group name.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -UseFailoverGroup <String>
        Specifies whether failover rules are automatically created, manually 
        created by the administrator, or disabled.  For FCP and iSCSI LIFs, the 
        default policy is 'disabled'; for NFS, CIFs and fcache LIFs, the 
        default policy is 'system_defined'.  Possible values: 
        
        "system_defined" - Use port roles to select failover targets
        "disabled"       - Failover disabled
        "enabled"        - Use a failover group to select failover targets
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -AutoRevert [<Boolean>]
        If true, data LIF will revert to its home node under certain 
        circumstances such as startup, and load balancing migration capability 
        is disabled automatically.
        
        Required?                    false
        Position?                    named
        Default value                false
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -AdministrativeStatus <String>
        Specifies the administrative status of the LIF.  The administrative 
        status can differ from the operational status; for instance, if you 
        specify the status as up but a network problem prevents the interface 
        from functioning, the operational status remains as down.  Possible 
        values: 'up', 'down', 'unknown'
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Comment <String>
        Comment to give the interface.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -NetmaskLength <Int32>
        The number of bits in the netmask.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -ListenForDnsQuery [<Boolean>]
        If true, this IP address will listen for DNS queries for the dns-zone 
        specified.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Subnet <String>
        Subnet where the interface address is allocated from.  If the option is 
        not used, the IP address will need to be provided by the administrator 
        during configuration.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -ForceSubnetAssociation
        If specified, force the LIF's subnet association.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Wwpn <String>
        The LIF's WWPN.
        
        This element is valid for FCP LIFs only.
        Note: This parameter is not available on Data ONTAP 8.4 and later.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -DnsUpdateEnabled [<Boolean>]
        Specifies whether DNS update is enabled for this LIF
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue)
        Accept wildcard characters?  false
        
    -Controller <NcController[]>
        A clustered Data ONTAP controller to receive this cmdlet, as embodied 
        by an NcController object.  This parameter is returned by the 
        Connect-NcController cmdlet.  If not specified, the value in the global 
        variable CurrentNcController is used.  In the latter case, if 
        CurrentNcController contains a collection of NcController objects, this 
        cmdlet is invoked on each controller in succession.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -ZapiRetryCount <Int32>
        The number of times to retry commands that return with errors that may 
        succeed after a retry
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216).
