Cmdlet: Set-NcSnapmirrorPolicy


NAME
    Set-NcSnapmirrorPolicy
    
SYNOPSIS
    Modify the specified SnapMirror Policy.
    
SYNTAX
    Set-NcSnapmirrorPolicy [-Name] <String> [-Tries <Int32>] [-UnlimitedTries] 
    [-TransferPriority <String>] [-Restart <String>] [-IgnoreAccessTime 
    [<Boolean>]] [-Comment <String>] [-EnableNetworkCompression [<Boolean>]] 
    [-WindowSizeForTdpMirror <String>] [-Owner <String>] [-SnapmirrorLabel 
    <String[]>] [-Keep <String[]>] [-Preserve <Boolean[]>] [-Warn <Int64[]>] 
    [-Schedule <String[]>] [-Prefix <String[]>] [-DiscardConfigs <String[]>] 
    [-VserverContext <String>] [-Controller <NcController[]>] [-ZapiRetryCount 
    <Int32>] [<CommonParameters>]
    
    
DESCRIPTION
    Modify the specified SnapMirror Policy.
    

PARAMETERS
    -Name <String>
        The name of the SnapMirror policy.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Tries <Int32>
        Specifies the maximum number of times to attempt each manual or 
        scheduled transfer for a SnapMirror relationship. The default is '8'.  
        This value is ignored if the UnlimitedTries switch is set.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -UnlimitedTries
        If specified, specify an unlimited number of tries.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -TransferPriority <String>
        Specifies the priority at which a SnapMirror transfer runs.
        
        Possible values:
        'normal' : By default every transfer in the system has this priority. 
        These transfers are scheduled before most 'low' priority transfers.
        'low'    : These transfers have least priority and usually are 
        scheduled after most 'normal' priority transfers.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Restart <String>
        Defines the behavior of SnapMirror if an interrupted transfer exists. 
        Applies to SnapMirror data protection relationship only. Possible 
        values: "always", "never", "default"
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -IgnoreAccessTime [<Boolean>]
        If true, incremental transfers will ignore files which have only their 
        access time changed. Applies to SnapMirror vault relationships only.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Comment <String>
        A human readable description associated with the SnapMirror policy.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -EnableNetworkCompression [<Boolean>]
        If true, network compression is enabled for transfers.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -WindowSizeForTdpMirror <String>
        Window size for TDP Mirror Transfer Operations. The default size is 2M
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Owner <String>
        The owner of the SnapMirror policy.
        
        This parameter is available with Data ONTAP 8.4 and up.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -SnapmirrorLabel <String[]>
        Specifies the Snapshot copy label.
        
        This parameter is available with Data ONTAP 8.4 and up.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Keep <String[]>
        Specifies the Snapshot copy retention count.
        
        This parameter is available with Data ONTAP 8.4 and up.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Preserve <Boolean[]>
        Specifies whether Snapshot copy preserve is enabled, which determines 
        the behavior when the Snapshot copy retention count is reached on the 
        SnapMirror Vault destination.
        
        This parameter is available with Data ONTAP 8.4 and up.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Warn <Int64[]>
        Specifies the warning threshold count.
        
        This parameter is available with Data ONTAP 8.4 and up.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Schedule <String[]>
        Specifies the Snapshot copy creation schedule name associated with the 
        rule.
        
        This parameter is available with Data ONTAP 8.4 and up.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Prefix <String[]>
        Specifies the Snapshot copy creation prefix name associated with the 
        rule.
        
        This parameter is available with Data ONTAP 8.4 and up.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -DiscardConfigs <String[]>
        Configurations Not Replicated During Identity Preserve Vserver DR.
        
        This parameter is available with Data ONTAP 8.4 and up.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -VserverContext <String>
        Vserver to which to direct this command.  If the currently connected 
        controller (as embodied by an NcController object) represents a 
        cluster, there are multiple means to address commands to one of its 
        vservers.  Either set the Vserver field on the NcController object to 
        direct all subsequent commands to that vserver, or supply the 
        VserverContext parameter to an individual cmdlet.  Only cmdlets which 
        may be invoked on vservers offer the VserverContext parameter.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <NcController[]>
        A clustered Data ONTAP controller to receive this cmdlet, as embodied 
        by an NcController object.  This parameter is returned by the 
        Connect-NcController cmdlet.  If not specified, the value in the global 
        variable CurrentNcController is used.  In the latter case, if 
        CurrentNcController contains a collection of NcController objects, this 
        cmdlet is invoked on each controller in succession.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -ZapiRetryCount <Int32>
        The number of times to retry commands that return with errors that may 
        succeed after a retry
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    DataONTAP.C.Types.SnapmirrorPolicy.SnapmirrorPolicyInfo
