Cmdlet: Set-NcSecuritySessionUserLimit


NAME
    Set-NcSecuritySessionUserLimit
    
SYNOPSIS
    Modifies a per-user session limit.
    
SYNTAX
    Set-NcSecuritySessionUserLimit [-Interface] <String> [-Username] <String> 
    [-Vserver] <String> [-MaxActiveLimit] <Int32> [-Controller 
    <NcController[]>] [-ZapiRetryCount <Int32>] [<CommonParameters>]
    
    
DESCRIPTION
    Modifies a per-user session limit.
    

PARAMETERS
    -Interface <String>
        The interface (cli or ontapi) for this application limit.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Username <String>
        The name of the authenticated user for this limit.
        
        Required?                    true
        Position?                    2
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Vserver <String>
        The name of the Vserver for this limit.
        
        Required?                    true
        Position?                    3
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -MaxActiveLimit <Int32>
        The maximum number of concurrent sessions allowed for this *
        
        Required?                    true
        Position?                    4
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Controller <NcController[]>
        A clustered Data ONTAP controller to receive this cmdlet, as embodied 
        by an NcController object.  This parameter is returned by the 
        Connect-NcController cmdlet.  If not specified, the value in the global 
        variable CurrentNcController is used.  In the latter case, if 
        CurrentNcController contains a collection of NcController objects, this 
        cmdlet is invoked on each controller in succession.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -ZapiRetryCount <Int32>
        The number of times to retry commands that return with errors that may 
        succeed after a retry
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    DataONTAP.C.Types.SecuritySession.SessionUserLimitsInfo
        
        
    
     
    
NOTES
    
    
        Category: security session
        API: security-session-limit-user-modify
        Family: cluster
    
    --------------  Example 1 --------------
    
    C:\PS>Set-NcSecuritySessionUserLimit -Interface cli -Username aparajir 
    -Vserver test -MaxActiveLimit 24
    
    
    Modifies the limit for maximum concurrent CLI sessions by user aparajir on 
    the test vserver
    
    
    
    Username                             Vserver                        
    Interface                            MaxActiveLimit
    --------                             -------                        
    ---------                            --------------
    aparajir                             test                           cli     
                                             24
