Cmdlet: Remove-NcHostVolume


NAME
    Remove-NaHostVolume
    
SYNOPSIS
    Remove a Windows volume from a disk.
    
SYNTAX
    Remove-NaHostVolume [[-Path] <String>] [-Reclaim] [-Force] [-UnmapSize 
    <String>] [<CommonParameters>]
    
    Remove-NaHostVolume [-VdsId] <Guid> [-Reclaim] [-Force] [-UnmapSize 
    <String>] [<CommonParameters>]
    
    
DESCRIPTION
    Remove a Windows volume from a disk.
    
    This cmdlet must be run with administrative privileges.
    

PARAMETERS
    -Path <String>
        The mount point (i.e. G:, 
        \\?\Volume{12345678-abcd-1234-abcd-1234567890ab}\) or a path to any 
        file on the volume to process
        
        Required?                    false
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Reclaim
        Free any blocks formerly occupied by the volume
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Force
        Continue even if volume is in use. This might cause unexpected errors 
        in any application using this volume.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -UnmapSize <String>
        
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -VdsId <Guid>
        The Guid object which uniquely identifies the volume in VDS.  Note that 
        this is not the same as the volume guid path.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216).
