Cmdlet: Get-NcVolMoveTargetAggr


NAME
    Get-NcVolMoveTargetAggr
    
SYNOPSIS
    Scans aggregates and returns a list of compatible target aggregates for the 
    given volume move operation.
    
SYNTAX
    Get-NcVolMoveTargetAggr [-Name] <String> [[-Vserver] <String>] 
    [-VserverContext <String>] [-Query <VolumeMoveTargetAggrInfo>] [-Attributes 
    <VolumeMoveTargetAggrInfo>] [-DestinationTieringPolicy <String>] 
    [-Controller <NcController[]>] [-ZapiRetryCount <Int32>] 
    [<CommonParameters>]
    
    Get-NcVolMoveTargetAggr [-VserverContext <String>] -Template [-Fill] 
    [-Controller <NcController[]>] [-ZapiRetryCount <Int32>] 
    [<CommonParameters>]
    
    
DESCRIPTION
    Scans aggregates and returns a list of compatible target aggregates for the 
    given volume move operation.
    

PARAMETERS
    -Name <String>
        The name of the volume to be moved.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Vserver <String>
        The name of the vserver that contains the volume to be moved.
        
        Required?                    false
        Position?                    2
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -VserverContext <String>
        Vserver to which to direct this command.  If the currently connected 
        controller (as embodied by an NcController object) represents a 
        cluster, there are multiple means to address commands to one of its 
        vservers.  Either set the Vserver field on the NcController object to 
        direct all subsequent commands to that vserver, or supply the 
        VserverContext parameter to an individual cmdlet.  Only cmdlets which 
        may be invoked on vservers offer the VserverContext parameter.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Query <VolumeMoveTargetAggrInfo>
        For advanced queries, provide a VolumeMoveTargetAggrInfo object with 
        various fields set to indicate which records to retrieve.  Other fields 
        should remain set to null.  Use "Get-NcVolMoveTargetAggr -Template" to 
        get the initially empty VolumeMoveTargetAggrInfo object.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Attributes <VolumeMoveTargetAggrInfo>
        For improved scalability in large clusters, provide a 
        VolumeMoveTargetAggrInfo object with various fields set to indicate 
        which fields to return on all objects.  Undesired fields should remain 
        set to null.  Use "Get-NcVolMoveTargetAggr -Template" to get the 
        initially empty VolumeMoveTargetAggrInfo object.  If not specified, all 
        data is returned for each object.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -DestinationTieringPolicy <String>
        Volume Tiering Policy.
        
        This parameter is available with Data ONTAP 9.2 and up.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Controller <NcController[]>
        A clustered Data ONTAP controller to receive this cmdlet, as embodied 
        by an NcController object.  This parameter is returned by the 
        Connect-NcController cmdlet.  If not specified, the value in the global 
        variable CurrentNcController is used.  In the latter case, if 
        CurrentNcController contains a collection of NcController objects, this 
        cmdlet is invoked on each controller in succession.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -ZapiRetryCount <Int32>
        The number of times to retry commands that return with errors that may 
        succeed after a retry
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Template
        Specify to get an empty VolumeMoveTargetAggrInfo object for use with 
        advanced queries involving the Query attribute.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Fill
        Specify when using the Template parameter to recursively fill in all 
        object types.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false

        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,

        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216).
