Cmdlet: New-NcExportRule


NAME
    New-NcExportRule
    
SYNOPSIS
    Create an export rule configuration.
    
SYNTAX
    New-NcExportRule [-Policy] <String> [[-Index] <UInt32>] -ClientMatch 
    <String> -ReadOnlySecurityFlavor <String[]> -ReadWriteSecurityFlavor 
    <String[]> [-Protocol <String[]>] [-Anon <String>] 
    [-SuperUserSecurityFlavor <String[]>] [-EnableSetUid] [-DisableSetUid] 
    [-EnableDev] [-DisableDev] [-NtfsUnixSecurityOps <String>] [-ChownMode 
    <String>] [-VserverContext <String>] [-Controller <NcController[]>] 
    [-ZapiRetryCount <Int32>] [<CommonParameters>]
    
    
DESCRIPTION
    Create an export rule configuration.
    

PARAMETERS
    -Policy <String>
        Export policy name.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Index <UInt32>
        Export rule index. A next available value is assigned if no value is 
        provided during create else the rule is inserted at the given rule 
        index position in the export-rule table.
        
        Required?                    false
        Position?                    2
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -ClientMatch <String>
        Client match specification for Export rule. The clients specified are 
        enforced with this Export rule. The rule with the higher rule index 
        value takes precedence.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -ReadOnlySecurityFlavor <String[]>
        Rule for read-only access.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -ReadWriteSecurityFlavor <String[]>
        Rule for read-write access.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Protocol <String[]>
        Client access protocol. Default value is 'any'.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Anon <String>
        User name or ID to which anonymous users are mapped. Default value is 
        '65534'.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -SuperUserSecurityFlavor <String[]>
        Security flavors for the superuser. Default value is 'never'.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -EnableSetUid
        If specified, the NFS server will honor SetUID bits in SETATTR 
        operation.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -DisableSetUid
        If specified, the NFS server will not honor SetUID bits in SETATTR 
        operation.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -EnableDev

        Required?                    false
        Position?                    named

        Accept pipeline input?       false
        Accept wildcard characters?  false

        If specified, the NFS server will not allow creation of devices.

        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -NtfsUnixSecurityOps <String>
        Ignore/Fail unix security operations on NTFS volumes. Possible values 
        are 'ignore', 'fail'. Default value is 'fail'.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -ChownMode <String>

        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false

        Vserver to which to direct this command.  If the currently connected 
        controller (as embodied by an NcController object) represents a 
        cluster, there are multiple means to address commands to one of its 
        vservers.  Either set the Vserver field on the NcController object to 
        direct all subsequent commands to that vserver, or supply the 
        VserverContext parameter to an individual cmdlet.  Only cmdlets which 
        may be invoked on vservers offer the VserverContext parameter.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false

        CurrentNcController contains a collection of NcController objects, this

        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
