Cmdlet: Invoke-NcSnapmirrorLsUpdate


NAME
    Invoke-NcSnapmirrorLsUpdate
    
SYNOPSIS
    Updates destination volumes of a set of load-sharing mirrors.
    
SYNTAX
    Invoke-NcSnapmirrorLsUpdate [-SourceCluster <String>] -SourceVserver 
    <String> -SourceVolume <String> [-Passthru] [-Controller <NcController[]>] 
    [-ZapiRetryCount <Int32>] [<CommonParameters>]
    
    Invoke-NcSnapmirrorLsUpdate [-Source] <String> [-Passthru] [-Controller 
    <NcController[]>] [-ZapiRetryCount <Int32>] [<CommonParameters>]
    
    
DESCRIPTION
    Updates destination volumes of a set of load-sharing mirrors.  The cmdlet 
    makes destination volumes in the group of load-sharing mirrors up-to-date 
    mirrors of the source volume.  Separate SnapMirror transfers are performed 
    from the source volume to each of the up-to-date destination volumes in the 
    set of load-sharing mirrors.  Load-sharing mirrors that lag behind the 
    up-to-date destination volumes might not be updated by the cmdlet.  Use 
    Invoke-NcSnapmirrorUpdate to update a lagging load-sharing mirror.  A job 
    will be spawned to operate on the snapmirror and the job id will be 
    returned. The progress of the job can be tracked using the job cmdlets.
    
    You must specify the source endpoint when using 
    Invoke-NcSnapmirrorLsInitialize.
