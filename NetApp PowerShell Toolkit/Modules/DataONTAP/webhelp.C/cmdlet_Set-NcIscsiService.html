Cmdlet: Set-NcIscsiService


NAME
    Set-NcIscsiService
    
SYNOPSIS
    Modify an iSCSI Service in a Vserver.
    
SYNTAX
    Set-NcIscsiService [-MaxErrorRecoveryLevel <Int32>] [-RetainTimeout 
    <Int32>] [-LoginTimeout <Int32>] [-MaxConnPerSession <Int32>] 
    [-MaxCmdsPerSession <Int32>] [-TcpWindowSize <Int32>] [-VserverContext 
    <String>] [-Controller <NcController[]>] [-ZapiRetryCount <Int32>] 
    [<CommonParameters>]
    
    Set-NcIscsiService [-Attributes] <IscsiServiceInfo> [-VserverContext 
    <String>] [-Controller <NcController[]>] [-ZapiRetryCount <Int32>] 
    [<CommonParameters>]
    
    
DESCRIPTION
    Modify an iSCSI Service in a Vserver.
    

PARAMETERS
    -MaxErrorRecoveryLevel <Int32>
        Maximum error recovery level. The possible values are:
        0 - session failure recovery.
        1 - digest failure recovery.
        2 - connection failure recovery.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -RetainTimeout <Int32>
        Specifies the wait time before active task reassignment is possible 
        after an unexpected connection termination.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -LoginTimeout <Int32>
        Specifies maximum time the login phase remains active until the iSCSI 
        target terminates the connection.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -MaxConnPerSession <Int32>
        Specifies the maximum number of connections per session that the iSCSI 
        target can accept.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -MaxCmdsPerSession <Int32>
        Specifies the maximum number of connections per session that the iSCSI 
        target can accept.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -TcpWindowSize <Int32>
        TCP receive window size in bytes.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -VserverContext <String>
        Vserver to which to direct this command.  If the currently connected 
        controller (as embodied by an NcController object) represents a 
        cluster, there are multiple means to address commands to one of its 
        vservers.  Either set the Vserver field on the NcController object to 
        direct all subsequent commands to that vserver, or supply the 
        VserverContext parameter to an individual cmdlet.  Only cmdlets which 
        may be invoked on vservers offer the VserverContext parameter.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <NcController[]>
        A clustered Data ONTAP controller to receive this cmdlet, as embodied 
        by an NcController object.  This parameter is returned by the 
        Connect-NcController cmdlet.  If not specified, the value in the global 
        variable CurrentNcController is used.  In the latter case, if 
        CurrentNcController contains a collection of NcController objects, this 
        cmdlet is invoked on each controller in succession.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -ZapiRetryCount <Int32>
        The number of times to retry commands that return with errors that may 
        succeed after a retry
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Attributes <IscsiServiceInfo>
        Specify an IscsiServiceInfo object from which to copy service data
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216).
