Cmdlet: ConvertTo-NcVhdx


NAME
    ConvertTo-NaVhdx
    
SYNOPSIS
    Convert a VMDK file, a fixed VHD file, or a Data ONTAP LUN into a VHDX file.
    
SYNTAX
    ConvertTo-NaVhdx [-SourceVhd] <String> [[-DestinationVhdx] <String>] 
    [-BlockSize <String>] [[-SnapshotName] <String>] [-Controller 
    <AbstractController>] [-SkipUnmap] [-UnmapSize <String>] 
    [<CommonParameters>]
    
    ConvertTo-NaVhdx -SourceVmdk <String> [[-DestinationVhdx] <String>] 
    [-BlockSize <String>] [-Win2k3ScsiToIde] [-FollowParent] [[-SnapshotName] 
    <String>] [-Controller <AbstractController>] [-SkipUnmap] [-UnmapSize 
    <String>] [<CommonParameters>]
    
    ConvertTo-NaVhdx -SourceLun <String> [[-DestinationVhdx] <String>] 
    [-BlockSize <String>] [[-SnapshotName] <String>] [-Controller 
    <AbstractController>] [-SkipUnmap] [-UnmapSize <String>] 
    [<CommonParameters>]
    
    ConvertTo-NaVhdx -SourceDiskIndex <UInt32> [[-DestinationVhdx] <String>] 
    [-BlockSize <String>] [[-SnapshotName] <String>] [-Controller 
    <AbstractController>] [-SkipUnmap] [-UnmapSize <String>] 
    [<CommonParameters>]
    
    
DESCRIPTION
    Convert a VMDK file, a fixed VHD file, or a Data ONTAP LUN into a VHDX file.
    
    A VMDK file must be of type "vmfs", "monolithicFlat", or 
    "twoGbMaxExtentFlat".
    
    When converting from a Data ONTAP LUN, the destination VHD file must reside 
    on a different LUN in the same volume.
    
    The source and destination files must reside inside of a Data ONTAP volume 
    (accessed using a path starting with /vol/<volname>), a Data ONTAP attached 
    LUN, or a Data ONTAP CIFS share (accessed using a mapped network drive or a 
    UNC path starting with \\hostname\<share>).  The source and destination 
    files must reside on the same controller and volume.
    
    If the controller supplied in the Controller parameter is not the 
    controller containing the files, the cmdlet consults the credentials cache 
    and attempts to connect via HTTPS.  If credentials are not present in the 
    credentials cache, an attempt to connect via RPC is made.  If a connection 
    cannot be made, the cmdlet will fail.
    

PARAMETERS
    -SourceVhd <String>
        Absolute path to a fixed VHD file to convert to a VHDX file.
        
        The path can be a path to a file on a Data ONTAP LUN, a path to a file 
        on a Data ONTAP CIFS share (either a mapped network drive, or a UNC 
        path starting with \\hostname\share), or a path to a file on a Data 
        ONTAP controller (starting with /vol/<volname/).
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue)
        Accept wildcard characters?  false
        
    -DestinationVhdx <String>
        Absolute path to the destination VHDX file.  The file must not yet 
        exist.  If not specified and the source is either a VHD or VMDK, the 
        destination VHDX will be created in the same directory alongside the 
        source.
        
        The path can be a path to a file on a Data ONTAP LUN, a path to a file 
        on a Data ONTAP CIFS share (either a mapped network drive, or a UNC 
        path starting with \\hostname\share), or a path to a file on a Data 
        ONTAP controller (starting with /vol/<volname/).
        
        Required?                    false
        Position?                    2
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -BlockSize <String>
        The block size to assign the created VHDX file.  The block size defines 
        the size of each payload chunk inside the VHDX.  The block size must be 
        a power of 2 and at least 1 MB and at most 256 MB.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -SnapshotName <String>
        The name of snapshot to be used during cloning.
        
        Required?                    false
        Position?                    3
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <AbstractController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -SkipUnmap
        
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -UnmapSize <String>
        
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -SourceVmdk <String>
        Path to the source VMDK file.  The VMDK file must be of type "vmfs", 
        "monolithicFlat", or "twoGbMaxExtentFlat".
        
        The path can be a path to a file on a Data ONTAP LUN, a path to a file 
        on a Data ONTAP CIFS share (either a mapped network drive, or a UNC 
        path starting with \\hostname\share), or a path to a file on a Data 
        ONTAP controller (starting with /vol/<volname/).
        
        Note:  This cmdlet can convert a VMDK that is being used by a running 
        VM.  Be sure to shutdown the VM or quiesce the filesystem before 
        running this cmdlet.  Failing to do so may result in an inconsistent or 
        corrupt result.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue)
        Accept wildcard characters?  false
        
    -Win2k3ScsiToIde
        Use this switch when converting a Windows Server 2003 SCSI VMDK to 
        VHDX.  When specified, the cmdlet will perform the necessary steps to 
        ensure the required IDE drivers are present on the virtual disk.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -FollowParent
        If specified, the parser will follow the VMDK parent descriptor and use 
        the base VMDK for the clone operation.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -SourceLun <String>
        Path of a LUN to convert to a VHDX file.  Must start with 
        /vol/<volname>/.  The following LUN types are supported: windows_2008, 
        hyper_v, linux, aix, hpux, netware, vmware, xen
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -SourceDiskIndex <UInt32>
        A Windows disk index of the disk to convert to a VHD.  The disk index 
        must reference a Data ONTAP LUN or the cmdlet will fail.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
