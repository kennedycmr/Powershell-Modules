Cmdlet: New-NcSecuritySessionAppLimit


NAME
    New-NcSecuritySessionAppLimit
    
SYNOPSIS
    Creates a per-application session limit.
    
SYNTAX
    New-NcSecuritySessionAppLimit [-Application] <String> [-Interface] <String> 
    [-MaxActiveLimit] <Int32> [-Controller <NcController[]>] [-ZapiRetryCount 
    <Int32>] [<CommonParameters>]
    
    
DESCRIPTION
    Creates a per-application session limit.
    

PARAMETERS
    -Application <String>
        The name of the application for this limit. Valid values are 
        '-default-', 'ssh', 'telnet', 'rsh', 'console', or 'ngsh'.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Interface <String>
        The interface (cli or ontapi) for this application limit.
        
        Required?                    true
        Position?                    2
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -MaxActiveLimit <Int32>
        The maximum number of concurrent sessions allowed for this application
        
        Required?                    true
        Position?                    3
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Controller <NcController[]>
        A clustered Data ONTAP controller to receive this cmdlet, as embodied 
        by an NcController object.  This parameter is returned by the 
        Connect-NcController cmdlet.  If not specified, the value in the global 
        variable CurrentNcController is used.  In the latter case, if 
        CurrentNcController contains a collection of NcController objects, this 
        cmdlet is invoked on each controller in succession.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -ZapiRetryCount <Int32>
        The number of times to retry commands that return with errors that may 
        succeed after a retry
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    DataONTAP.C.Types.SecuritySession.SessionApplicationLimitsInfo
        
        
    
     
    
NOTES
    
    
        Category: security session
        API: security-session-limit-application-create
        Family: cluster
    
    --------------  Example 1 --------------
    
    C:\PS>New-NcSecuritySessionAppLimit -Application telnet  -Interface cli 
    -MaxActiveLimit 128
    
    
    Creates a limit of 128 active telnet clients for the CLI
    
    
    
    Application                                                         
    Interface                            MaxActiveLimit
    -----------                                                         
    ---------                            --------------
    telnet                                                              cli     
                                            128
