Cmdlet: ConvertTo-NcLun


NAME
    ConvertTo-NaLun
    
SYNOPSIS
    Create a LUN from a virtual hard disk (VHD or VHDX).
    
SYNTAX
    ConvertTo-NaLun [-SourceVhd] <String> [-DestinationLun] <String> 
    [-Reserved] [[-Type] <String>] [[-SnapshotName] <String>] [-Controller 
    <AbstractController>] [<CommonParameters>]
    
    ConvertTo-NaLun -SourceVhdx <String> [-DestinationLun] <String> [-Reserved] 
    [[-Type] <String>] [[-SnapshotName] <String>] [-Controller 
    <AbstractController>] [<CommonParameters>]
    
    
DESCRIPTION
    Create a LUN from a virtual hard disk (VHD or VHDX).  The virtual hard disk 
    file must reside on a Data ONTAP LUN.  The path of the LUN to create must 
    lie on the same volume as the LUN containing the VHD file or the cmdlet 
    will fail.
    
    If the controller supplied in the Controller parameter is not the 
    controller containing the VHD file, the cmdlet consults the credentials 
    cache and attempts to connect via HTTPS.  If credentials are not present in 
    the credentials cache, an attempt to connect via RPC is made.  If a 
    connection cannot be made, the cmdlet will fail.
    

PARAMETERS
    -SourceVhd <String>
        Virtual hard disk file to create a new LUN from.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -DestinationLun <String>
        The path to give the new LUN.  The new LUN must reside in the same 
        volume as the LUN containing the source VHD.
        
        Required?                    true
        Position?                    2
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Reserved
        If specified, the created LUN will have space reservation enabled.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Type <String>
        OS type to give the created LUN.  If not specified, the created LUN 
        will have the type "windows_2008".  Issue "help New-NaLun -Parameter 
        Type" for more information on LUN OS types.  The following types are 
        supported: windows_2008, hyper_v, linux, aix, hpux, netware, vmware, xen
        
        Required?                    false
        Position?                    3
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -SnapshotName <String>
        The name of snapshot to be used during cloning.
        
        Required?                    false
        Position?                    4
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <AbstractController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -SourceVhdx <String>
        VHDX file from which to create a LUN.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216).
