Cmdlet: Get-NcClusterPeerHealth


NAME
    Get-NcClusterPeerHealth
    
SYNOPSIS
    Get the health of all local and peer nodes sequentially.
    
SYNTAX
    Get-NcClusterPeerHealth [[-Name] <String[]>] [-Node <String[]>] 
    [-IsClusterHealthy [<Boolean>]] [-IsNodeHealthy [<Boolean>]] 
    [-IsNodeAvailable [<Boolean>]] [-Attributes <ClusterPeerHealthInfo>] 
    [-Controller <NcController[]>] [-ZapiRetryCount <Int32>] 
    [<CommonParameters>]
    
    Get-NcClusterPeerHealth -Template [-Controller <NcController[]>] 
    [-ZapiRetryCount <Int32>] [<CommonParameters>]
    
    Get-NcClusterPeerHealth -Query <ClusterPeerHealthInfo> [-Attributes 
    <ClusterPeerHealthInfo>] [-Controller <NcController[]>] [-ZapiRetryCount 
    <Int32>] [<CommonParameters>]
    
    
DESCRIPTION
    Get the health of all local and peer nodes sequentially.
    

PARAMETERS
    -Name <String[]>
        The name of the peer cluster.  Data ONTAP wildcards are permitted.
        
        Required?                    false
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Node <String[]>
        The name of the node in the peer cluster.  Data ONTAP wildcards are 
        permitted.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -IsClusterHealthy [<Boolean>]
        The status of peer cluster health.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -IsNodeHealthy [<Boolean>]
        The RDB health of the node.  True means the node is in quorum.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -IsNodeAvailable [<Boolean>]
        True implies that the destination node is available for intercluster 
        communication.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Attributes <ClusterPeerHealthInfo>
        For improved scalability in large clusters, provide a 
        ClusterPeerHealthInfo object with various fields set to indicate which 
        fields to return on all objects.  Undesired fields should remain set to 
        null.  Use "Get-NcClusterPeerHealth -Template" to get the initially 
        empty ClusterPeerHealthInfo object.  If not specified, all data is 
        returned for each object.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <NcController[]>
        A clustered Data ONTAP controller to receive this cmdlet, as embodied 
        by an NcController object.  This parameter is returned by the 
        Connect-NcController cmdlet.  If not specified, the value in the global 
        variable CurrentNcController is used.  In the latter case, if 
        CurrentNcController contains a collection of NcController objects, this 
        cmdlet is invoked on each controller in succession.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -ZapiRetryCount <Int32>
        
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Template
        Specify to get an empty ClusterPeerHealthInfo object for use with 
        advanced queries involving the Query attribute.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Query <ClusterPeerHealthInfo>
        For advanced queries, provide a ClusterPeerHealthInfo object with 
        various fields set to indicate which records to retrieve.  Other fields 
        should remain set to null.  Use "Get-NcClusterPeerHealth -Template" to 
        get the initially empty ClusterPeerHealthInfo object.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216).
