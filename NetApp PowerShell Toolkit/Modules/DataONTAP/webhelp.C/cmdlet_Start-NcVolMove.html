Cmdlet: Start-NcVolMove


NAME
    Start-NcVolMove
    
SYNOPSIS
    Moves a flexvol to any eligible aggregate in the cluster.
    
SYNTAX
    Start-NcVolMove [-Name] <String> [-DestinationAggregate] <String> [-Vserver 
    <String>] [-CutoverWindow <Int32>] [-CutoverAttempts <Int32>] 
    [-CutoverAction <String>] [-ValidateOnly] [-EncryptDestination] 
    [-DecryptDestination] [-GenerateDestinationKey] [-TieringPolicy <String>] 
    [-Controller <NcController[]>] [-ZapiRetryCount <Int32>] 
    [<CommonParameters>]
    
    
DESCRIPTION
    Moves a flexvol to any eligible aggregate in the cluster.  The list of 
    eligible aggregates can be obtained using Get-NcVolMoveTargetAggr.  This 
    API will start the move.  It will run a series of checks to determine if 
    the volume can be moved.  If any of the checks results in an error or 
    warning, the API will return with an error. The user has to take the 
    necessary corrective action before restarting the move.  If all the checks 
    pass, the API will return a success.  If ValidateOnly is not specified, the 
    destination volume will be created and the move will start.  By default, 
    the move will cutover automatically.  A background job will be created and 
    the API returns immediately with the job ID information. The status of the 
    move job can be obtained by using Get-NcJob.

    -CutoverAction <String>
        Specifies the action to be taken for cutover. Possible values are 
        "abort_on_failure", "defer_on_failure", "force" and "wait". Default is 
        "defer_on_failure". If "abort_on_failure" action is specified, the job 
        will try cutover till the cutover attempts are exhausted, and cleanup 
        if it fails to cutover.  If "defer_on_failure" action is specified, the 
        job will try cutover till the cutover attempts are exhausted and move 
        into "cutover deferred" state if it fails to cutover.  This is the 
        default option.  The volume move job waits for intervention to restart 
        the cutover process again.  If "force" action is specified, the job 
        will try cutover till the cutover attempts are exhausted and force the 
        cutover at the expense of disrupting the clients.  If "wait" action is 
        specified, when the job hits the decision point, it will not go into 
        cutover automatically, instead it will wait for the user to issue 
        Invoke-NcVolMoveCutover as the signal to try the cutover.
        
        Required?                    false
        Position?                    named
        Default value                defer_on_failure
        Accept pipeline input?       false
        Accept wildcard characters?  false
