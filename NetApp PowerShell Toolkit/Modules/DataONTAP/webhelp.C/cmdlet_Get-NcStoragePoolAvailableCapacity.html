Cmdlet: Get-NcStoragePoolAvailableCapacity


NAME
    Get-NcStoragePoolAvailableCapacity
    
SYNOPSIS
    Get spare capacity available on each node for storage pools.
    
SYNTAX
    Get-NcStoragePoolAvailableCapacity [[-StoragePool] <String[]>] [[-Node] 
    <String[]>] [-Attributes <StoragePoolAvailableCapacityInfo>] [-Controller 
    <NcController[]>] [-ZapiRetryCount <Int32>] [<CommonParameters>]
    
    Get-NcStoragePoolAvailableCapacity -Template [-Controller <NcController[]>] 
    [-ZapiRetryCount <Int32>] [<CommonParameters>]
    
    Get-NcStoragePoolAvailableCapacity -Query 
    <StoragePoolAvailableCapacityInfo> [-Attributes 
    <StoragePoolAvailableCapacityInfo>] [-Controller <NcController[]>] 
    [-ZapiRetryCount <Int32>] [<CommonParameters>]
    
    
DESCRIPTION
    Get spare capacity available on each node for storage pools.
    

PARAMETERS
    -StoragePool <String[]>
        Name of one or more shared storage pools.  Data ONTAP wildcards are 
        permitted.
        
        Required?                    false
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Node <String[]>
        Name of one or more nodes.  Data ONTAP wildcards are permitted.
        
        Required?                    false
        Position?                    2
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Attributes <StoragePoolAvailableCapacityInfo>
        For improved scalability in large clusters, provide a 
        StoragePoolAvailableCapacityInfo object with various fields set to 
        indicate which fields to return on all objects.  Undesired fields 
        should remain set to null.  Use "Get-NcStoragePoolAvailableCapacity 
        -Template" to get the initially empty StoragePoolAvailableCapacityInfo 
        object.  If not specified, all data is returned for each object.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <NcController[]>
        A clustered Data ONTAP controller to receive this cmdlet, as embodied 
        by an NcController object.  This parameter is returned by the 
        Connect-NcController cmdlet.  If not specified, the value in the global 
        variable CurrentNcController is used.  In the latter case, if 
        CurrentNcController contains a collection of NcController objects, this 
        cmdlet is invoked on each controller in succession.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -ZapiRetryCount <Int32>
        
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Template
        Specify to get an empty StoragePoolAvailableCapacityInfo object for use 
        with advanced queries involving the Query attribute.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Query <StoragePoolAvailableCapacityInfo>
        For advanced queries, provide a StoragePoolAvailableCapacityInfo object 
        with various fields set to indicate which records to retrieve.  Other 
        fields should remain set to null.  Use 
        "Get-NcStoragePoolAvailableCapacity -Template" to get the initially 
        empty StoragePoolAvailableCapacityInfo object.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
