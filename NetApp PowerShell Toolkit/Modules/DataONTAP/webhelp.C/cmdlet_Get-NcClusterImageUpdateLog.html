Cmdlet: Get-NcClusterImageUpdateLog


NAME
    Get-NcClusterImageUpdateLog
    
SYNOPSIS
    Get details from the update log
    
SYNTAX
    Get-NcClusterImageUpdateLog [-ComponentId <String[]>] [-TransactionId 
    <Int64>] [-Attributes <ClusterUpdateTransactionLogInfo>] [-Controller 
    <NcController[]>] [-ZapiRetryCount <Int32>] [<CommonParameters>]
    
    Get-NcClusterImageUpdateLog -Template [-Controller <NcController[]>] 
    [-ZapiRetryCount <Int32>] [<CommonParameters>]
    
    Get-NcClusterImageUpdateLog -Query <ClusterUpdateTransactionLogInfo> 
    [-Attributes <ClusterUpdateTransactionLogInfo>] [-Controller 
    <NcController[]>] [-ZapiRetryCount <Int32>] [<CommonParameters>]
    
    
DESCRIPTION
    Get details from the update log
    

PARAMETERS
    -ComponentId <String[]>
        Restrict results to specific components (like a node) by specifying 
        their IDs (node name).
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -TransactionId <Int64>
        Get the log of one specific transaction using the transaction ID
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Attributes <ClusterUpdateTransactionLogInfo>
        For improved scalability in large clusters, provide a 
        ClusterUpdateTransactionLogInfo object with various fields set to 
        indicate which fields to return on all objects.  Undesired fields 
        should remain set to null.  
        
        Use "Get-NcClusterImageUpdateLog -Template" to get the initially empty 
        ClusterUpdateTransactionLogInfo object.  If not specified, all data is 
        returned for each object.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <NcController[]>
        A clustered Data ONTAP controller to receive this cmdlet, as embodied 
        by an NcController object.  This parameter is returned by the 
        Connect-NcController cmdlet.  If not specified, the value in the global 
        variable CurrentNcController is used.  In the latter case, if 
        CurrentNcController contains a collection of NcController objects, this 
        cmdlet is invoked on each controller in succession.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -ZapiRetryCount <Int32>
        The number of times to retry commands that return with errors that may 
        succeed after a retry
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Template
        Specify to get an empty ClusterUpdateTransactionLogInfo object for use 
        with advanced queries involving the Query attribute.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Query <ClusterUpdateTransactionLogInfo>
        For advanced queries, provide a ClusterUpdateTransactionLogInfo object 
        with various fields set to indicate which records to retrieve.  Other 
        fields should remain set to null.  Use "Get-NcClusterImageUpdateLog 
        -Template" to get the initially empty ClusterUpdateTransactionLogInfo 
        object.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216).
