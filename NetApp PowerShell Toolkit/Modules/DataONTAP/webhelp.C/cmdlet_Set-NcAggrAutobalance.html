Cmdlet: Set-NcAggrAutobalance


NAME
    Set-NcAggrAutobalance
    
SYNOPSIS
    Modify the Auto Balance Aggregate configuration.
    
SYNTAX
    Set-NcAggrAutobalance [-MoveMode <String>] [-StatePollingInterval <Int32>] 
    [-UnbalancedThresholdPercent <Int32>] [-AvailableThresholdPercent <Int32>] 
    [-VolumeActivityThreshold <Int32>] [-VolumeInactiveDelayThreshold <Int32>] 
    [-HoldDurationAfterMove <Int32>] [-Controller <NcController[]>] 
    [-ZapiRetryCount <Int32>] [<CommonParameters>]
    
    
DESCRIPTION
    Modify the Auto Balance Aggregate configuration.
    

PARAMETERS
    -MoveMode <String>
        The mode dictates what Auto Balance Aggregate will do with the 
        suggested moves. Possible values: 'recommend', 'auto'.
        
        In 'recommend' mode, the suggested moves will be displayed, but no 
        further action will be taken. In 'auto' mode, Auto Balance Aggregate 
        will trigger moves to start. The default mode is 'recommend'.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -StatePollingInterval <Int32>
        Polling period, in seconds, used by Auto Balance Aggregate to update 
        volume and aggregate stats. The default value is 360 seconds (1 hour).
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -UnbalancedThresholdPercent <Int32>
        This percentage will indicate the space used threshold that will cause 
        Auto Balance Aggregate to consider this aggregate as unbalanced. The 
        default value is 60%.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -AvailableThresholdPercent <Int32>
        When Auto Balance Aggregate identifies an aggregate that needs to be 
        balanced, it will attempt to identify other target aggregates it can 
        move volumes to. 
        
        An aggregate will only be considered as a target if its projected space 
        usage after the move is still under this percentage. Auto Balance 
        Aggregate will attempt to move volumes from an unbalanced aggregate 
        until it is under this percentage. 
        
        The default value is 40%.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -VolumeActivityThreshold <Int32>
        The threshold, in number of operations per second preformed on a volume 
        (read/write/internal), that will disqualify the volume from being a 
        candidate to be moved by Auto Balance Aggregate. 
                
        The default value is 100 operations per second.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -VolumeInactiveDelayThreshold <Int32>
        This value represents a duration in seconds and designates the amount 
        of time that has to elapse after a volume has exceeded the 
        VolumeActivityThreshold before it can be considered a movable volume by 
        Auto Balance Aggregate.
        
        The default value is 24 hours.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -HoldDurationAfterMove <Int32>
        This value represents the number of seconds that need to have elapsed 
        since the volume was last moved before it will be considered a movable 
        candidate again.
        
        The default value is 48 hours.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Controller <NcController[]>
        A clustered Data ONTAP controller to receive this cmdlet, as embodied 
        by an NcController object.  This parameter is returned by the 
        Connect-NcController cmdlet.  If not specified, the value in the global 
        variable CurrentNcController is used.  In the latter case, if 
        CurrentNcController contains a collection of NcController objects, this 
        cmdlet is invoked on each controller in succession.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -ZapiRetryCount <Int32>
        
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
