Cmdlet: Add-NcFileDirectorySecurityPolicyTask


NAME
    Add-NcFileDirectorySecurityPolicyTask
    
SYNOPSIS
    Add a file security policy task.
    
SYNTAX
    Add-NcFileDirectorySecurityPolicyTask [-Name] <String> [-Path] <String> 
    [-SecurityType <String>] [-NtfsMode <String>] [-NtfsSecurityDescriptor 
    <String>] [-Index <Int32>] [-AccessControl <String>] [-VserverContext 
    <String>] [-Controller <NcController[]>] [-ZapiRetryCount <Int32>] 
    [<CommonParameters>]
    
    Add-NcFileDirectorySecurityPolicyTask [-Name] <String> [-Path] <String> 
    [-SecurityType <String>] [-NtfsMode <String>] [-NtfsSecurityDescriptor 
    <String>] [-Index <Int32>] [-AccessControl <String>] -PassThru 
    [-VserverContext <String>] [-Controller <NcController[]>] [-ZapiRetryCount 
    <Int32>] [<CommonParameters>]
    
    
DESCRIPTION
    Add a file security policy task.  A task maps security descriptor with a 
    file or folder.
    

PARAMETERS
    -Name <String>
        The name of the policy to which the task needs to be added.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Path <String>
        The file or folder path of the task.
        
        Required?                    true
        Position?                    2
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -SecurityType <String>
        The type of security.  Possible values: "ntfs", "nfsv4"
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -NtfsMode <String>
        NTFS propagation mode.  Possible values: "propagate", "ignore", 
        "replace"
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -NtfsSecurityDescriptor <String>
        NTFS security descriptor identifier.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Index <Int32>
        the target index/position of this task in the policy.  If a policy has 
        already 5 tasks and this is the 6th task you are adding and you want to 
        add this task as 2nd task, you can specifiy the index 2. By default the 
        task gets added as last task. If the index number exceeds the number of 
        positions, it will go at the end.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -AccessControl <String>
        Specifies the access control of task to be applied. Possible values: 
        "file-directory", "slag"
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -VserverContext <String>
        Vserver to which to direct this command.  If the currently connected 
        controller (as embodied by an NcController object) represents a 
        cluster, there are multiple means to address commands to one of its 
        vservers.  Either set the Vserver field on the NcController object to 
        direct all subsequent commands to that vserver, or supply the 
        VserverContext parameter to an individual cmdlet.  Only cmdlets which 
        may be invoked on vservers offer the VserverContext parameter.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <NcController[]>
        A clustered Data ONTAP controller to receive this cmdlet, as embodied 
        by an NcController object.  This parameter is returned by the 
        Connect-NcController cmdlet.  If not specified, the value in the global 
        variable CurrentNcController is used.  In the latter case, if 
        CurrentNcController contains a collection of NcController objects, this 
        cmdlet is invoked on each controller in succession.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -ZapiRetryCount <Int32>
        
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -PassThru
        If specified, output an object representing the NTFS security policy.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
