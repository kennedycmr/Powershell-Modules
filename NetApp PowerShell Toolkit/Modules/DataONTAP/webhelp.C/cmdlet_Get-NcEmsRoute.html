Cmdlet: Get-NcEmsRoute


NAME
    Get-NcEmsRoute
    
SYNOPSIS
    Get information on EMS routes.
    
SYNTAX
    Get-NcEmsRoute [[-Name] <String[]>] [[-Severity] <String[]>] [-Attributes 
    <EmsRoutingInfo>] [-Controller <NcController[]>] [-ZapiRetryCount <Int32>] 
    [<CommonParameters>]
    
    Get-NcEmsRoute -Template [-Controller <NcController[]>] [-ZapiRetryCount 
    <Int32>] [<CommonParameters>]
    
    Get-NcEmsRoute -Query <EmsRoutingInfo> [-Attributes <EmsRoutingInfo>] 
    [-Controller <NcController[]>] [-ZapiRetryCount <Int32>] 
    [<CommonParameters>]
    
    
DESCRIPTION
    Get information on EMS routes.
    

PARAMETERS
    -Name <String[]>
        The name of the EMS message.  Data ONTAP wildcards are permitted.
        
        Required?                    false
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Severity <String[]>
        The event severity.  Data ONTAP wildcards are permitted.  Possible 
        values:
            "emergency"      - System is unusable
            "alert"          - Action must be taken immediately
            "critical"       - Critical condition
            "error"          - Error condition
            "warning"        - Warning condition
            "notice"         - Normal but significant condition
            "informational"  - Information message
            "debug"          - A debugging message
        
        Required?                    false
        Position?                    2
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Attributes <EmsRoutingInfo>
        For improved scalability in large clusters, provide an EmsRoutingInfo 
        object with various fields set to indicate which fields to return on 
        all objects.  Undesired fields should remain set to null.  Use 
        "Get-NcEmsRoute -Template" to get the initially empty EmsRoutingInfo 
        object.  If not specified, all data is returned for each object.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <NcController[]>
        A clustered Data ONTAP controller to receive this cmdlet, as embodied 
        by an NcController object.  This parameter is returned by the 
        Connect-NcController cmdlet.  If not specified, the value in the global 
        variable CurrentNcController is used.  In the latter case, if 
        CurrentNcController contains a collection of NcController objects, this 
        cmdlet is invoked on each controller in succession.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -ZapiRetryCount <Int32>
        
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Template
        Specify to get an empty EmsRoutingInfo object for use with advanced 
        queries involving the Query attribute.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Query <EmsRoutingInfo>
        For advanced queries, provide a EmsRoutingInfo object with various 
        fields set to indicate which records to retrieve.  Other fields should 
        remain set to null.  Use "Get-NcEmsRoute -Template" to get the 
        initially empty EmsRoutingInfo object.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    DataONTAP.C.Types.Ems.EmsRoutingInfo
