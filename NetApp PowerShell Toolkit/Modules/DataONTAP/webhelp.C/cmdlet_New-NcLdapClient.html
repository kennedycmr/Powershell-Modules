Cmdlet: New-NcLdapClient


NAME
    New-NcLdapClient
    
SYNOPSIS
    Create a new Lightweight Directory Access Protocol (LDAP) client 
    configuration for the cluster.
    
SYNTAX
    New-NcLdapClient [-Name] <String> [-Schema] <String> [-AdDomain <String>] 
    [-PreferredAdServers <String[]>] [-DoNotBindAsCifsServer] [-TcpPort 
    <Int32>] [-QueryTimeout <Int32>] [-MinBindLevel <String>] [-BindDn 
    <String>] [-BindPassword <String>] [-BaseDn <String>] [-BaseScope <String>] 
    [-UserDn <String>] [-UserScope <String>] [-GroupDn <String>] [-GroupScope 
    <String>] [-NetGroupDn <String>] [-NetGroupScope <String>] [-UseStartTls] 
    [-VserverContext <String>] [-IsNetgroupByHostEnabled [<Boolean>]] 
    [-NetgroupByHostScope <String>] [-AllowSsl] [-SessionSecurity <String>] 
    [-ObsPassword <String>] [-Controller <NcController[]>] [-ZapiRetryCount 
    <Int32>] [<CommonParameters>]
    
    New-NcLdapClient [-Name] <String> [-Schema] <String> [-Servers <String[]>] 
    [-TcpPort <Int32>] [-QueryTimeout <Int32>] [-MinBindLevel <String>] 
    [-BindDn <String>] [-BindPassword <String>] [-BaseDn <String>] [-BaseScope 
    <String>] [-UserDn <String>] [-UserScope <String>] [-GroupDn <String>] 
    [-GroupScope <String>] [-NetGroupDn <String>] [-NetGroupScope <String>] 
    [-UseStartTls] [-VserverContext <String>] [-IsNetgroupByHostEnabled 
    [<Boolean>]] [-NetgroupByHostScope <String>] [-AllowSsl] [-SessionSecurity 
    <String>] [-ObsPassword <String>] [-Controller <NcController[]>] 
    [-ZapiRetryCount <Int32>] [<CommonParameters>]
    
    
DESCRIPTION
    Create a new Lightweight Directory Access Protocol (LDAP) client 
    configuration for the cluster.
    
    The cmdlet works in the cluster context for Data ONTAP versions prior to 
    8.2.  In Data ONTAP 8.2 and later, the cmdlet works in the vserver context.

        Position?                    1
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Schema <String>

        can be obtained using the Get-NcLdapClientSchema cmdlet.
        
        Required?                    true
        Position?                    2
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -AdDomain <String>

    -PreferredAdServers <String[]>
        Preferred Active Directory (AD) Domain controllers to use for this

        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -DoNotBindAsCifsServer
        If specified, the cluster will not use the CIFS server's credentials to

        Required?                    false
        Position?                    named

        Accept pipeline input?       true (ByPropertyName)

    -TcpPort <Int32>
        The TCP port on the LDAP server to use for this configuration. If 
        omitted, this parameter defaults to 389.

        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -MinBindLevel <String>
        The minimum authentication level that can be used to authenticate with 
        the LDAP server. If omitted, this parameter defaults to 'sasl' if the 
        configuration uses Active Directory LDAP. For configurations that use 
        LDAP servers from other vendors, this parameter defaults to 'simple' if 
        BindDn is specified and 'anonymous' otherwise. Possible values: 
        "anonymous" - Anonymous bind
        "simple"    - Simple bind
        "sasl"      - Simple Authentication and Security Layer (SASL) bind

        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -BindDn <String>
        The Bind Distinguished Name (DN) is the LDAP identity used during the 
        authentication process by the clients. This is required if the LDAP 
        server does not support anonymous binds. This field is not used if 
        BindAsCifsServer is specified. Example : 
        cn=username,cn=Users,dc=example,dc=com
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -BindPassword <String>

        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -BaseDn <String>
        Indicates the starting point for searches within the LDAP directory 
        tree. If omitted, searches will start at the root of the directory tree.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -BaseScope <String>
        This indicates the scope for LDAP search. If omitted, this parameter 
        defaults to 'subtree'. Possible values:
        "base"      - Search only the base directory entry
        "onelevel"  - Search the base directory entry and the children of the 
        base entry
        "subtree"   - Search the base directory entry and all its descendants
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -UserDn <String>
        The User Distinguished Name (DN), if specified, is used as the starting

        Position?                    named

        Accept wildcard characters?  false

        "base"      - Search only the base directory entry
        "onelevel"  - Search the base directory entry and the children of the

        "subtree"   - Search the base directory entry and all its descendants.

        Position?                    named
        Default value

        Accept wildcard characters?  false

        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)

        Accept pipeline input?       true (ByPropertyName)

        Position?                    named
