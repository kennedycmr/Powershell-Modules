Cmdlet: Set-NcNetPortBroadcastDomain


NAME
    Set-NcNetPortBroadcastDomain
    
SYNOPSIS
    Modify the attributes of the layer 2 broadcast domain object.
    
SYNTAX
    Set-NcNetPortBroadcastDomain [-Name] <String> [[-Ipspace] <String>] [-Mtu] 
    <Int32> [-Controller <NcController[]>] [-ZapiRetryCount <Int32>] 
    [<CommonParameters>]
    
    Set-NcNetPortBroadcastDomain [-Name] <String> [[-Ipspace] <String>] 
    -AddPort <String[]> [-Controller <NcController[]>] [-ZapiRetryCount 
    <Int32>] [<CommonParameters>]
    
    Set-NcNetPortBroadcastDomain [-Name] <String> [[-Ipspace] <String>] 
    -RemovePort <String[]> [-Controller <NcController[]>] [-ZapiRetryCount 
    <Int32>] [<CommonParameters>]
    
    
DESCRIPTION
    Modify the attributes of the layer 2 broadcast domain object.
    

PARAMETERS
    -Name <String>
        Name of the layer 2 broadcast domain.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Ipspace <String>
        IPspace name that the layer 2 broadcast domain belongs to.  The default 
        value for this parameter is 'Default'.
        
        Required?                    false
        Position?                    2
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Mtu <Int32>
        Maximum Transmission Unit (MTU) of the layer 2 broadcast domain. The 
        MTU will be applied to all ports that belong to the layer 2 broadcast 
        domain.
        
        Required?                    true
        Position?                    3
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Controller <NcController[]>
        A clustered Data ONTAP controller to receive this cmdlet, as embodied 
        by an NcController object.  This parameter is returned by the 
        Connect-NcController cmdlet.  If not specified, the value in the global 
        variable CurrentNcController is used.  In the latter case, if 
        CurrentNcController contains a collection of NcController objects, this 
        cmdlet is invoked on each controller in succession.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -ZapiRetryCount <Int32>
        
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -AddPort <String[]>
        Ports to be added to the broadcast domain.
        
        Format:node-name:port-name
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -RemovePort <String[]>
        Ports to be removed from the broadcast domain.
        
        Format:node-name:port-name
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    DataONTAP.C.Types.Net.NetPortBroadcastDomainInfo

    --------------  Example 1 --------------
    
    C:\PS>Set-NcNetPortBroadcastDomain -Name test -Ipspace test -Mtu 1500
    
    
    Change the MTU for the given net port broadcast domain.
    
    
    
    BroadcastDomain            Ipspace              FailoverGroups            
    Mtu PortUpdateStatusCombined
    ---------------            -------              --------------            
    --- ------------------------
    test                       test                 {test}                   
    1500 complete
    
    
    
    --------------  Example 2 --------------
    
    C:\PS>Set-NcNetPortBroadcastDomain -Name Default -AddPort beam-fs-ps-01:e0d
    
    
    Add the specified port to the net port broadcast domain.
    
    
    
    BroadcastDomain            Ipspace              FailoverGroups            
    Mtu PortUpdateStatusCombined
    ---------------            -------              --------------            
    --- ------------------------
    Default                    Default              {Default}                
    1500 complete
    
    
    
    --------------  Example 3 --------------
    
    C:\PS>Set-NcNetPortBroadcastDomain -Name pester_test -RemovePort 
    beam-fs-ps-01:e0b -Ipspace test
    
    
    Remove the given port from the net port broadcast domain.
    
    
    
    BroadcastDomain            Ipspace              FailoverGroups            
    Mtu PortUpdateStatusCombined
    ---------------            -------              --------------            
    --- ------------------------
    pester_test                test                                          
    1500 complete
