Cmdlet: Get-NcDiagnosisAlert


NAME
    Get-NcDiagnosisAlert
    
SYNOPSIS
    Get a list of Alert Information objects.
    
SYNTAX
    Get-NcDiagnosisAlert [[-Node] <String[]>] [[-Monitor] <String[]>] 
    [[-AlertId] <String[]>] [[-AlertingResource] <String[]>] [-Attributes 
    <DiagnosisAlertInfo>] [-Controller <NcController[]>] [-ZapiRetryCount 
    <Int32>] [<CommonParameters>]
    
    Get-NcDiagnosisAlert -Template [-Controller <NcController[]>] 
    [-ZapiRetryCount <Int32>] [<CommonParameters>]
    
    Get-NcDiagnosisAlert -Query <DiagnosisAlertInfo> [-Attributes 
    <DiagnosisAlertInfo>] [-Controller <NcController[]>] [-ZapiRetryCount 
    <Int32>] [<CommonParameters>]
    
    
DESCRIPTION
    Get a list of Alert Information objects.
    

PARAMETERS
    -Node <String[]>
        One or more nodes hosting the health monitor.  Data ONTAP wildcards are 
        permitted.
        
        Required?                    false
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Monitor <String[]>
        One or more health monitors.  Data ONTAP wildcards are permitted.
        
        Required?                    false
        Position?                    2
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -AlertId <String[]>
        One or more alert ids.  Data ONTAP wildcards are permitted.
        
        Required?                    false
        Position?                    3
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -AlertingResource <String[]>
        One or more unique names of the resource that generated the alert.  
        Data ONTAP wildcards are permitted.
        
        Required?                    false
        Position?                    4
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Attributes <DiagnosisAlertInfo>
        For improved scalability in large clusters, provide a 
        DiagnosisAlertInfo object with various fields set to indicate which 
        fields to return on all objects.  Undesired fields should remain set to 
        null.  Use "Get-NcDiagnosisAlert -Template" to get the initially empty 
        DiagnosisAlertInfo object.  If not specified, all data is returned for 
        each object.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <NcController[]>
        A clustered Data ONTAP controller to receive this cmdlet, as embodied 
        by an NcController object.  This parameter is returned by the 
        Connect-NcController cmdlet.  If not specified, the value in the global 
        variable CurrentNcController is used.  In the latter case, if 
        CurrentNcController contains a collection of NcController objects, this 
        cmdlet is invoked on each controller in succession.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -ZapiRetryCount <Int32>
        
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Template
        Specify to get an empty DiagnosisAlertInfo object for use with advanced 
        queries involving the Query attribute.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Query <DiagnosisAlertInfo>
        For advanced queries, provide a DiagnosisAlertInfo object with various 
        fields set to indicate which records to retrieve.  Other fields should 
        remain set to null.  Use "Get-NcDiagnosisAlert -Template" to get the 
        initially empty DiagnosisAlertInfo object.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    DataONTAP.C.Types.Diagnosis.DiagnosisAlertInfo
        
        
    
     
    
NOTES
    
    
        Category: diagnosis
        API: diagnosis-alert-get-iter
        Family: cluster
    
    --------------  Example 1 --------------
    
    C:\PS>Get-NcDiagnosisAlert -Node VxeRubble-01
    
    
    Get all of the diagnosis alerts for node VxeRubble-01.
    
    
    
    Monitor          AlertId                        AlertingResource         
    Subsystem    PerceivedSeverity  Node
    -------          -------                        ----------------         
    ---------    -----------------  ----
    node_connect     DualPathToDiskShelf_Alert      50:05:0c:c1:02:01:6e:43  
    sas_connect  major              VxeRubble-01
    node_connect     DualPathToDiskShelf_Alert      50:05:0c:c1:02:01:63:d2  
    sas_connect  major              VxeRubble-01

    C:\PS>$q = Get-NcDiagnosisAlert -Template

    Get-NcDiagnosisAlert -Query $q
