Cmdlet: ConvertTo-NcVmdk


NAME
    ConvertTo-NaVmdk
    
SYNOPSIS
    Convert a VHD or VHDX file into a VMDK file.
    
SYNTAX
    ConvertTo-NaVmdk [-SourceVhd] <String> [[-DestinationVmdk] <String>] 
    [[-VmdkType] <VmdkDescriptor+CreateType>] [[-AdapterType] 
    <VmdkDescriptor+AdapterType>] [[-VirtualHwVersion] <Byte>] [-SnapshotName 
    <String>] [-Controller <AbstractController>] [-SkipUnmap] [-UnmapSize 
    <String>] [<CommonParameters>]
    
    ConvertTo-NaVmdk [-SourceVhdx] <String> [[-DestinationVmdk] <String>] 
    [[-VmdkType] <VmdkDescriptor+CreateType>] [[-AdapterType] 
    <VmdkDescriptor+AdapterType>] [[-VirtualHwVersion] <Byte>] [-SnapshotName 
    <String>] [-Controller <AbstractController>] [-SkipUnmap] [-UnmapSize 
    <String>] [<CommonParameters>]
    
    
DESCRIPTION
    Convert a VHD or VHDX file into a VMDK file.
    
    The source and destination files must reside inside of a Data ONTAP volume 
    (accessed using a path starting with /vol/<volname>), a Data ONTAP attached 
    LUN, or a Data ONTAP CIFS share (accessed using a mapped network drive or a 
    UNC path starting with \\hostname\<share>).  The source and destination 
    files must reside on the same controller and volume.
    
    If the controller supplied in the Controller parameter is not the 
    controller containing the files, the cmdlet consults the credentials cache 
    and attempts to connect via HTTPS.  If credentials are not present in the 
    credentials cache, an attempt to connect via RPC is made.  If a connection 
    cannot be made, the cmdlet will fail.
    

PARAMETERS
    -SourceVhd <String>
        Path to the source VHD file.  The path can be a path to the file on a 
        Data ONTAP mounted LUN or a path to a Data ONTAP CIFS share (either a 
        mapped network drive or a UNC path starting with \\hostname\<share>).
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -DestinationVmdk <String>
        Path to the destination VMDK file.  If not supplied, the VMDK file will 
        be created in the same directory as the source VMDK.  The file must not 
        exist.  The path can be a path to the file on the controller (starting 
        with /vol/<volname>), a path to the file on a Data ONTAP mounted LUN, 
        or a path to a Data ONTAP CIFS share (either a mapped network drive or 
        a UNC path starting with \\hostname\<share>).
        
        Required?                    false
        Position?                    2
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -VmdkType <VmdkDescriptor+CreateType>
        The VMDK type of the VMDK file to be created.  The supported types are: 
         "vmfs", "monolithicFlat", and "twoGbMaxExtentFlat".  If not provided, 
        the type will be "vmfs".
        
        Required?                    false
        Position?                    3
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -AdapterType <VmdkDescriptor+AdapterType>
        The adapter type to assign the created VMDK.  Supported values types 
        are: "lsilogic", "buslogic", and "ide".  If not provided, the type will 
        be "lsilogic".
        
        Required?                    false
        Position?                    4
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -VirtualHwVersion <Byte>
        Specify the virtual hardware version of the virtual machine which will 
        use this disk.
        
        Default is 7 (supported on ESX/ESXi 4.x and later).
        
        Required?                    false
        Position?                    5
        Default value                7
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -SnapshotName <String>
        The name of snapshot to be used during cloning.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <AbstractController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -SkipUnmap
        
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -UnmapSize <String>
        
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -SourceVhdx <String>
        Path to the source VHDX file.  The path can be a path to the file on a 
        Data ONTAP mounted LUN, a path to a Data ONTAP CIFS share (either a 
        mapped network drive or a UNC path starting with \\hostname\<share>), 
        or a path to a file residing on a Data ONTAP controller (starting with 
        /vol/<volname>/).
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216).
