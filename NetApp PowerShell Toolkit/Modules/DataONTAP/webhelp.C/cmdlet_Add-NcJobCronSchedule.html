Cmdlet: Add-NcJobCronSchedule


NAME
    Add-NcJobCronSchedule
    
SYNOPSIS
    Create a new cron job schedule entry.
    
SYNTAX
    Add-NcJobCronSchedule [-Name] <String> [-Month <Int64[]>] [-Day <Int64[]>] 
    [-DayOfWeek <Int64[]>] [-Hour <Int64[]>] [-Minute <Int64[]>] 
    [-JobScheduleCluster <String>] [-Controller <NcController[]>] 
    [-ZapiRetryCount <Int32>] [<CommonParameters>]
    
    
DESCRIPTION
    Create a new cron job schedule entry.
    

PARAMETERS
    -Name <String>
        The name of the cron job schedule to create.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Month <Int64[]>
        The month(s) when the job should be run.  -1 represents all months.  
        Zero represents January.  Range: [-1..11].
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Day <Int64[]>
        The day(s) of the month when the job should be run.  -1 represents all 
        days of a month from 1 to 31.  Range: [-1..31].
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -DayOfWeek <Int64[]>
        The day(s) in the week when the job should be run.  -1 represents all 
        days of a week.  Zero represents Sunday.  Range: [-1..6].
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Hour <Int64[]>
        The hour(s) of the day when the job should be run.  -1 represents all 
        hours.  Range: [-1..23].
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Minute <Int64[]>
        The minute(s) of each hour when the job should be run.  -1 represents 
        all minutes.  Range: [-1..59].
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -JobScheduleCluster <String>
        The cluster owner of the job schedule. Defaults to the local cluster. 
        In a Metrocluster configuration, the partner cluster can be specified.
        
        This parameter is available with DataONTAP 9.3 and up
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Controller <NcController[]>
        A clustered Data ONTAP controller to receive this cmdlet, as embodied 
        by an NcController object.  This parameter is returned by the 
        Connect-NcController cmdlet.  If not specified, the value in the global 
        variable CurrentNcController is used.  In the latter case, if 
        CurrentNcController contains a collection of NcController objects, this 
        cmdlet is invoked on each controller in succession.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -ZapiRetryCount <Int32>
        
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    DataONTAP.C.Types.Job.JobScheduleCronInfo
        
        
    
     
    
NOTES
    
    
        Category: job
        API: job-schedule-cron-create
        Family: cluster
    
    --------------  Example 1 --------------
    
    C:\PS>Add-NcJobCronSchedule 15min -Minute 0,15,30,45
    
    
    Define a cron schedule entry for jobs that should run every 15 minutes.
    
    
    
    JobScheduleName           JobScheduleDescription
    ---------------           ----------------------
    15min                     @:00,:15,:30,:45
