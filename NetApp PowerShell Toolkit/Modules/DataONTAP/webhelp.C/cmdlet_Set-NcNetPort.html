Cmdlet: Set-NcNetPort


NAME
    Set-NcNetPort
    
SYNOPSIS
    Modify the attributes of network port.
    
SYNTAX
    Set-NcNetPort [-Name] <String> [-Node] <String> [-Role <String>] [-Mtu 
    <Int32>] [-AutoNegotiate [<Boolean>]] [-Duplex <String>] [-Speed <String>] 
    [-FlowControl <String>] [-Up] [-Down] [-AutoRevertDelay <Int32>] [-Ipspace 
    <String>] [-PortType <String>] [-InterfaceGroupNode <String>] 
    [-InterfaceGroupPort <String>] [-InterfaceGroupDistributionFunction 
    <String>] [-InterfaceGroupMode <String>] [-MtuAdmin <Int64>] 
    [-IgnoreHealthStatus] [-Controller <NcController[]>] [-ZapiRetryCount 
    <Int32>] [<CommonParameters>]
    
    
DESCRIPTION
    Modify the attributes of network port.
    

PARAMETERS
    -Name <String>
        Specifies the name of port.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Node <String>
        Specifies the name of the node that owns the port.
        
        Required?                    true
        Position?                    2
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Role <String>
        Specifies the role associated with the port.  Possible values:
        
        "undef"          - No defined role
        "cluster"        - Used for communication using the private cluster 
        network
        "data"           - Used for communicating with file service clients
        "node_mgmt"      - Used by administrators to configure the node
        "intercluster"   - Used for communication with a different cluster
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Mtu <Int32>
        Specifies the maximum transmission unit (MTU) of the port.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -AutoNegotiate [<Boolean>]
        Enables or disables ethernet auto-negotiation of speed, duplex and flow 
        control.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Duplex <String>
        Specifies the user preferred duplex setting of the port.  Possible 
        values:
        
        "undef"     - No defined duplex
        "auto"      - Auto-negotiate duplex setting for link
        "half"      - Half-duplex link usage
        "full"      - Full-duplex link usage
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Speed <String>
        Specifies the user preferred speed setting of the port.  Possible 
        values:
        
        "undef"     - No defined speed
        "auto"      - Auto-negotiate speed for link
        "10"        - 10 megabits per second
        "100"       - 100 megabits per second
        "1000"      - 1 gigabit per second
        "10000"     - 10 gigabits per second
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -FlowControl <String>
        Specifies the user preferred flow control setting of the port.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Up
        If specified, set the state of the port to 'up'.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Down
        If specified, set the state of the port to 'down'.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -AutoRevertDelay <Int32>
        For a port with role 'cluster', specifies the delay in seconds before 
        autoreverting a LIF to this port.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Ipspace <String>
        Specifies the port's associated IPspace name. Note: the 'Cluster' 
        ipspace is reserved for cluster ports.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -PortType <String>
        Specifies the type of port.  Possible values:
        
        "physical"  - Physical network interface
        "if_group"  - Logical interface group (IEEE 802.3, section 43)
        "vlan"      - Virtual LAN (IEEE 802.1Q)
        "undef"     - No defined port type
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -InterfaceGroupNode <String>
        Specifies the name of the node on which interface group (ifgrp) is 
        created.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -InterfaceGroupPort <String>
        For a port of type 'ifgrp', specifies the name of the physical port on 
        which ifgrp is created.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -InterfaceGroupDistributionFunction <String>
        For a port of type 'ifgrp', specifies the traffic distribution 
        function.  Possible values:
        
        "mac"            - Network traffic is distributed on the basis of MAC 
        addresses
        "ip"             - Network traffic is distributed on the basis of IP 
        addresses
        "sequential"     - Network traffic is distributed round-robin to each 
        interface
        "port"           - Network traffic is distributed by transport layer 
        address 4-tuple
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -InterfaceGroupMode <String>
        For a port of type 'ifgrp', specifies the link policy.  Possible values:
        
        "multimode"      - All links are simultaneously active
        "multimode_lacp" - Link state is managed by the switch using link 
        aggregation control protocol (LACP) (IEEE 802.3ad)
        "singlemode"     - Only one link is active at a time
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -MtuAdmin <Int64>
        The user preferred maximum transmission unit (MTU) of the port.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -IgnoreHealthStatus
        If specified, the system ignores the specified port's health status.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Controller <NcController[]>
        A clustered Data ONTAP controller to receive this cmdlet, as embodied 
        by an NcController object.  This parameter is returned by the 
        Connect-NcController cmdlet.  If not specified, the value in the global 
        variable CurrentNcController is used.  In the latter case, if 
        CurrentNcController contains a collection of NcController objects, this 
        cmdlet is invoked on each controller in succession.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -ZapiRetryCount <Int32>
        The number of times to retry commands that return with errors that may 
        succeed after a retry
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    DataONTAP.C.Types.Net.NetPortInfo
        
        
    
     
    
NOTES
    
    
        Category: net
        API: net-port-modify

    --------------  Example 1 --------------
    
    C:\PS>Get-NcNetPort e2c -Node * | Set-NcNetPort -Up
    
    
    Get the 'e2c' ports on all nodes and set their administrative status to 
    'up'.
