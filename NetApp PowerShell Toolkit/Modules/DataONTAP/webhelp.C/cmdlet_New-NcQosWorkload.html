Cmdlet: New-NcQosWorkload


NAME
    New-NcQosWorkload
    
SYNOPSIS
    Create a new workload.
    
SYNTAX
    New-NcQosWorkload [-Name] <String> [-Controller <NcController[]>] 
    [-ZapiRetryCount <Int32>] [<CommonParameters>]
    
    
DESCRIPTION
    Create a new workload.  A unique name must be provided for the new 
    workload.  No work will be classified to a workload unless the workload has 
    at least one stream.  Therefore at least one stream should usually be 
    created along with a workload.  If the performance of the workload is to be 
    controlled by QoS, a policy must also be defined and attached to the 
    workload.
    
    This cmdlet is not available for Data ONTAP 8.2 and up.
    

PARAMETERS
    -Name <String>
        The user visible name of the QoS workload.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Controller <NcController[]>
        A clustered Data ONTAP controller to receive this cmdlet, as embodied 
        by an NcController object.  This parameter is returned by the 
        Connect-NcController cmdlet.  If not specified, the value in the global 
        variable CurrentNcController is used.  In the latter case, if 
        CurrentNcController contains a collection of NcController objects, this 
        cmdlet is invoked on each controller in succession.
        
        Required?                    false
        Position?                    named

        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -ZapiRetryCount <Int32>
        
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
