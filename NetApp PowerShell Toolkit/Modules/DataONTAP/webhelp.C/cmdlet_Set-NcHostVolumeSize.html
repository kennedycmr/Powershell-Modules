Cmdlet: Set-NcHostVolumeSize


NAME
    Set-NaHostVolumeSize
    
SYNOPSIS
    Grow or shrink a Windows volume.
    
SYNTAX
    Set-NaHostVolumeSize [[-Path] <String>] [-Size <String>] [-Minimum] 
    [-Maximum] [<CommonParameters>]
    
    Set-NaHostVolumeSize [-VdsId] <Guid> [-Size <String>] [-Minimum] [-Maximum] 
    [<CommonParameters>]
    
    
DESCRIPTION
    Grow or shrink a Windows volume.  This is done non-disruptively on active 
    volumes. Note that in some cases due to unmovable files or parent disk 
    size, it may not be possible to grow or shrink as much as you would like.
    
    This cmdlet must be run with administrative privileges and is only 
    supported on Windows Vista sp1, Server 2008 or later.
    

PARAMETERS
    -Path <String>
        The mount point (i.e. G:, 
        \\?\Volume{12345678-abcd-1234-abcd-1234567890ab}\) or a path to any 
        file on the volume to process
        
        Required?                    false
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Size <String>
        Shrink or grow the volume by a specific size. Do not use with -Minimum 
        or -Maximum.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Minimum
        Shrink the volume to the minimum size possible. Do not use with -Size 
        or -Maximum.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Maximum
        Grow the volume to the maximum size for the parent disk or LUN. Do not 
        use with -Size or -Minimum
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -VdsId <Guid>
        The Guid object which uniquely identifies the volume in VDS.  Note that 
        this is not the same as the volume guid path.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    DataONTAP.PowerShell.SDK.Cmdlets.Toolkit.Windows.HostVolume
        
        
    
     
    
NOTES
    
    
        Category: host
    
    --------------  Example 1 --------------
    
    C:\PS>Set-NaHostVolumeSize E:\ -Minimum
    
    
    Shrink the E:\ volume to the minimum possible size.
    
    
    
    
    
    --------------  Example 2 --------------
    
    C:\PS>Set-NaHostVolumeSize E:\ -Size "+50%"
    
    
    Grow the E:\Volume by 50%.
