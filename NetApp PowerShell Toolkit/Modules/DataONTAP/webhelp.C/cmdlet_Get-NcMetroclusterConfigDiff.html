Cmdlet: Get-NcMetroclusterConfigDiff


NAME
    Get-NcMetroclusterConfigDiff
    
SYNOPSIS
    Get the differences in configuration between two clusters in DR 
    relationship for MetroCluster.
    
SYNTAX
    Get-NcMetroclusterConfigDiff [[-SourceCluster] <String[]>] 
    [[-DestinationCluster] <String[]>] [-Object <String[]>] [-Instance 
    <UInt64[]>] [-Keys <String[]>] [-Attributes <ConfigClusterDiffInfo>] 
    [-Controller <NcController[]>] [-ZapiRetryCount <Int32>] 
    [<CommonParameters>]
    
    Get-NcMetroclusterConfigDiff -Template [-Controller <NcController[]>] 
    [-ZapiRetryCount <Int32>] [<CommonParameters>]
    
    Get-NcMetroclusterConfigDiff -Query <ConfigClusterDiffInfo> [-Attributes 
    <ConfigClusterDiffInfo>] [-Controller <NcController[]>] [-ZapiRetryCount 
    <Int32>] [<CommonParameters>]
    
    
DESCRIPTION
    Get the differences in configuration between two clusters in DR 
    relationship for MetroCluster. If an object is not specified, the 
    differences for all replicated objects are shown. If the source 
    configuration matches that of the destination, EOBJECTNOTFOUND is returned, 
    indicating that the difference is nil. This cmdlet can be invoked on the 
    source cluster or destination cluster.  
    
    Note: The configuration objects replicated as part of MetroCluster and 
    their keys can potentially change from release to release. If a program has 
    opted to use those as inputs for selection, it may have to be adjusted for 
    updated versions of Data ONTAP.
