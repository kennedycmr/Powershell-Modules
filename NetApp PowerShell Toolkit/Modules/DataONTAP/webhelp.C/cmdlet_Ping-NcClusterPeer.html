Cmdlet: Ping-NcClusterPeer


NAME
    Ping-NcClusterPeer
    
SYNOPSIS
    Ping the specified peer nodes.
    
SYNTAX
    Ping-NcClusterPeer [[-Name] <String[]>] [-Address <String[]>] [-PingStatus 
    <String[]>] [-RoundTrip <String[]>] [-Count <Int32>] [-Type <String>] 
    [-Timeout <Int32>] [-Attributes <ClusterPeerPingInfo>] [-Controller 
    <NcController[]>] [-ZapiRetryCount <Int32>] [<CommonParameters>]
    
    Ping-NcClusterPeer [-Count <Int32>] [-Type <String>] [-Timeout <Int32>] 
    -Template [-Controller <NcController[]>] [-ZapiRetryCount <Int32>] 
    [<CommonParameters>]
    
    Ping-NcClusterPeer [-Count <Int32>] [-Type <String>] [-Timeout <Int32>] 
    -Query <ClusterPeerPingInfo> [-Attributes <ClusterPeerPingInfo>] 
    [-Controller <NcController[]>] [-ZapiRetryCount <Int32>] 
    [<CommonParameters>]
    
    
DESCRIPTION
    Ping the specified peer nodes.  Ping operations take place between a node 
    in a cluster and a node in a remote cluster.  The originating node is the 
    node which initiates the ping operation and the destination cluster and 
    node specify the remote node which is the target of the ping.
    

PARAMETERS

        The name of the destination cluster.  Data ONTAP wildcards are 
        permitted.

        Default value

        Accept wildcard characters?  false

        The active IP address of the node to ping.  Data ONTAP wildcards are

        Position?                    named

        Accept wildcard characters?  false
        
    -PingStatus <String[]>

        Required?                    false

    -RoundTrip <String[]>

        ONTAP wildcards are permitted.
        
        Required?                    false
        Position?                    named
        Default value

    -Count <Int32>

        The network protocol to use when performing the ping operation.  Values

        Required?                    false

        Position?                    named
