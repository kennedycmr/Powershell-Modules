Cmdlet: New-NcVserver


NAME
    New-NcVserver
    
SYNOPSIS
    Create a new vserver.
    
SYNTAX
    New-NcVserver [-Name] <String> [[-RootVolume] <String>] 
    [[-RootVolumeAggregate] <String>] [[-NameServerSwitch] <String[]>] 
    [[-RootVolumeSecurityStyle] <String>] [-AntivirusOnAccessPolicy <String>] 
    [-Comment <String>] [-Language <String>] [-NameMappingSwitch <String[]>] 
    [-QuotaPolicy <String>] [-SnapshotPolicy <String>] [-RepositoryVserver] 
    [-Ipspace <String>] [-Subtype <String>] [-CachingPolicy <String>] [-Async] 
    [-Controller <NcController[]>] [-ZapiRetryCount <Int32>] 
    [<CommonParameters>]
    
    
DESCRIPTION
    Create a new vserver.
    

PARAMETERS
    -Name <String>
        The name of the new vserver.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -RootVolume <String>
        Root volume of the vserver.  This volume will be created automatically.
        
        Required?                    false
        Position?                    2
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -RootVolumeAggregate <String>
        The aggregate on which the root volume will be created.
        
        Required?                    false
        Position?                    3
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -NameServerSwitch <String[]>
        Name server switch configuration details for the vserver.  Possible 
        values: 'nis', 'file', 'ldap'.
        
        Required?                    false
        Position?                    4
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -RootVolumeSecurityStyle <String>
        Security style of the root volume.  Possible values: 'unix', 'ntfs', 
        'mixed'.
        
        Required?                    false
        Position?                    5
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -AntivirusOnAccessPolicy <String>
        Antivirus policy.  Default: default
        
        Required?                    false
        Position?                    named
        Default value                default
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Comment <String>
        Comment.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Language <String>
        Specifies the language to use for the vserver.  Default: C.
        
        Available language codes are: 
        
        'C' .............. POSIX 
        'ar' ............. Arabic 
        'cs' ............. Czech 
        'da' ............. Danish 
        'de' ............. German 
        'en' ............. English 
        'en_US' .......... English (US) 
        'es' ............. Spanish 
        'fi' ............. Finnish 
        'fr' ............. French 
        'he' ............. Hebrew 
        'hr' ............. Croatian 
        'hu' ............. Hungarian 
        'it' ............. Italian 
        'ja' ............. Japanese euc-j* 
        'ja_v1' .......... Japanese euc-j 
        'ja_JP.PCK' ...... Japanese PCK (sjis)* 
        'ja_JP.932' ...... Japanese cp932* 
        'ja_JP.PCK_v2' ... Japanese PCK (sjis) 
        'ko' ............. Korean 
        'no' ............. Norwegian 
        'nl' ............. Dutch 
        'pl' ............. Polish 
        'pt' ............. Portuguese 
        'ro' ............. Romanian 
        'ru' ............. Russian 
        'sk' ............. Slovak 
        'sl' ............. Slovenian 
        'sv' ............. Swedish 
        'tr' ............. Turkish 
        'zh' ............. Simplified Chinese 
        'zh.GBK' ......... Simplified Chinese (GBK) 
        'zh_TW' .......... Traditional Chinese euc-tw 
        'zh_TW.BIG5' ..... Traditional Chinese Big 5
        
        To use UTF-8 as the NFS character set, append '.UTF-8'.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -NameMappingSwitch <String[]>
        Name mapping switch configuration details for the vserver.  Possible 
        values: 'file', 'ldap'.  Default: file
        
        Required?                    false
        Position?                    named
        Default value                file
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -QuotaPolicy <String>
        Quota policy.  Default: default
        
        Required?                    false
        Position?                    named
        Default value                default
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -SnapshotPolicy <String>
        Snapshot policy setting.  Default: default
        
        Required?                    false
        Position?                    named
        Default value                default
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -RepositoryVserver
        If specified, create a repository Vserver. A repository Vserver 
        contains a single InfiniteVol. Once a repository Vserver is created in 
        a cluster,  additional  Vservers cannot be created. Conversely, a 
        repository   Vserver cannot be created if non-repository Vservers 
        already exist in a cluster.
        
        This option is only available in Data ONTAP 8.1.1 and up.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Ipspace <String>
        Ipspace name.  Default is 'Default'
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Subtype <String>
        Vserver Subtype. This option allows the admin to specify the sub-type 
        of Vserver being created.
        
        The possible options for this value are: default, dp-destination, 
        sync-source, sync-destination
        
        If a Vserver is an MCC destination, then it would have the sub-type 
        sync-source. Vserver of sub-type sync-destination cannot be explicitly 
        created by the admin. When a Vserver of sub-type dp-destination is 
        created, most of the management activities are not allowed. Default 
        value is default.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -CachingPolicy <String>
        The caching policy associated with the vserver, used to make caching 
        decisions. If a caching policy is not assigned to this Vserver, the 
        default cluster-wide policy is used.
        
        Accepted values are -
         'none' - Does not cache any user data or metadata blocks
         'auto' - Read caches all metadata blocks and randomly read user data 
        blocks; write caches all randomly overwritten user data blocks
         'meta' - Read caches only metadata blocks.
         'random_read' - Read caches all metadata and randomly read user data 
        blocks.
         'random_read_write' -  Read caches all metadata, randomly read and 
        randomly written user data blocks.
         'all_read' - Read caches all metadata, randomly read and sequentially 
        read user data blocks.
         'all_read_random_write' - Read caches all metadata, randomly read, 
        sequentially read and randomly written user data.
         'all' - Read caches all data blocks read and written. It does not do 
        any write caching.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Async
        Specify to perform vserver creation asynchronously.
        
        This operation is supported only on Data ONTAP 8.4 and up. If this 
        parameter is specified against an unsupported controller, the cmdlet 
        will throw an 'Invalid Argument' error.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <NcController[]>
        A clustered Data ONTAP controller to receive this cmdlet, as embodied 
        by an NcController object.  This parameter is returned by the 
        Connect-NcController cmdlet.  If not specified, the value in the global 
        variable CurrentNcController is used.  In the latter case, if 
        CurrentNcController contains a collection of NcController objects, this 
        cmdlet is invoked on each controller in succession.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -ZapiRetryCount <Int32>
        The number of times to retry commands that return with errors that may 
        succeed after a retry.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
