Cmdlet: Add-NaIscsiInitiatorAuth


NAME
    Add-NaIscsiInitiatorAuth
    
SYNOPSIS
    Add initiator to the authentication list.
    
SYNTAX
    Add-NaIscsiInitiatorAuth [-Initiator] <String> [-None] [-Controller 
    <NaController>] [<CommonParameters>]
    
    Add-NaIscsiInitiatorAuth [-Initiator] <String> -Chap [-Username <String>] 
    [-Password <String>] [-InboundCredential <PSCredential>] [-OutboundUsername 
    <String>] [-OutboundPassword <String>] [-OutboundCredential <PSCredential>] 
    [-Controller <NaController>] [<CommonParameters>]
    
    Add-NaIscsiInitiatorAuth [-Initiator] <String> -Deny [-Controller 
    <NaController>] [<CommonParameters>]
    
    
DESCRIPTION
    Add initiator to the authentication list.
    

PARAMETERS
    -Initiator <String>
        Name of initiator. i.e. iqn.1987-06.com.initvendor1:appsrv.sn.2346.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -None
        Specify this switch parameter for authentication type 'none'.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Chap
        Specify this switch parameter for authentication type 'CHAP'.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Username <String>
        Inbound CHAP user name.  For the CHAP authentication type, the 
        -Username and -Password or -InboundCredential parameters must be 
        specified.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Password <String>
        Inbound CHAP user password. For the CHAP authentication type, the 
        -Username and -Password or -InboundCredential parameters must be 
        specified.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -InboundCredential <PSCredential>
        PSCredential object containing inbound CHAP credentials.  The username 
        and password supplied in the credential object take precedence over the 
        username and password supplied by the -Username and -Password 
        parameters.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -OutboundUsername <String>
        Outbound CHAP user name.  Outbound authentication is optional. If 
        Outbound authentication is not specified, then the initiator can only 
        do inbound traffic.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -OutboundPassword <String>
        Outbound CHAP user password.  Outbound authentication is optional. If 
        Outbound authentication is not specified, then the initiator can only 
        do inbound traffic.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -OutboundCredential <PSCredential>
        PSCredential object containing outbound CHAP credentials.  The username 
        and password supplied in the credential object take precedence over the 
        username and password supplied by the -OutboundUsername and 
        -OutboundPassword parameters.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Deny
        Specify this switch parameter for authentication type 'deny'.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216).
