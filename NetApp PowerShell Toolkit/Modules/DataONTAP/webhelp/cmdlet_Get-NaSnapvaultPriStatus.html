Cmdlet: Get-NaSnapvaultPriStatus


NAME
    Get-NaSnapvaultPriStatus
    
SYNOPSIS
    Request the primary to return the status entries for desired relationships.
    
SYNTAX
    Get-NaSnapvaultPriStatus [[-Path] <String>] [-Controller <NaController>] 
    [<CommonParameters>]
    
    
DESCRIPTION
    Request the primary to return the status entries for desired relationships. 
    The relationships whose status are desired must be specified using the 
    system path. When snapvault primary and secondary are licensed on the same 
    filer, output of this API is identical to the output of 
    Get-NaSnapvaultSecStatus.
    

PARAMETERS
    -Path <String>
        System path for relationships whose status is desired.  Wildcards are 
        permitted.
        
        Required?                    false
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    DataONTAP.Types.Snapvault.SnapvaultStatusInfo
