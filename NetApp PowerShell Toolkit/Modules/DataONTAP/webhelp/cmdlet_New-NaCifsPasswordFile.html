Cmdlet: New-NaCifsPasswordFile


NAME
    New-NaCifsPasswordFile
    
SYNOPSIS
    Creates a basic /etc/passwd file including a root user with the specified 
    password.
    
SYNTAX
    New-NaCifsPasswordFile [[-Password] <String>] [-Credential <PSCredential>] 
    [-Controller <NaController>] [<CommonParameters>]
    
    
DESCRIPTION
    Creates a basic /etc/passwd file including a root user with the specified 
    password. Note: This will overwrite an existing /etc/passwd file.
    

PARAMETERS
    -Password <String>
        The password given to a locally defined root user which may be used in 
        the event that other authentication services are unavailable. This will 
        be defined in the filer's local /etc/passwd file.  Specifying this 
        parameter will trigger a password change for the root user.
        
        Required?                    false
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Credential <PSCredential>
        Credential containing the password given to a locally defined root user 
        which may be used in the event that other authentication services are 
        unavailable. This will be defined in the filer's local /etc/passwd 
        file.  Specifying this parameter will trigger a password change for the 
        root user.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216).
