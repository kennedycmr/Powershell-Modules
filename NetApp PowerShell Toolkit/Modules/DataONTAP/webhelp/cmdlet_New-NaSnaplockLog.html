Cmdlet: New-NaSnaplockLog


NAME
    New-NaSnaplockLog
    
SYNOPSIS
    Open a new WORM log file.  This will archive the active WORM log file.
    
SYNTAX
    New-NaSnaplockLog [[-Volume] <String>] [[-Basename] <String>] [-Controller 
    <NaController>] [<CommonParameters>]
    
    
DESCRIPTION
    Open a new WORM log file.  This will archive the active WORM log file.
    

PARAMETERS
    -Volume <String>
        The name of the volume that contains the WORM log files. This volume 
        must be a SnapLock Compliance volume. If the log volume is not 
        specified, the log volume is considered to be the volume configured by 
        the Set-NaSnaplockLogVolume command.
        
        Required?                    false
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Basename <String>
        The basename of the WORM log file to be archived. If the log basename 
        is not specified, all the active WORM log files in the log volume are 
        archived. If the old log file was basename.<old-date>-present, once the 
        archive command is run, the file will be renamed to 
        basename.<old-date>-<current-date> and closed. A new log file of the 
        name basename.<current-date>-present will be opened for future log 
        updates.
        
        Required?                    false
        Position?                    2
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    DataONTAP.Types.Snaplock.LogFileInfo
        
        
    
     
    
NOTES
    
    
        Category: snaplock
        API: snaplock-log-archive
    
    --------------  Example 1 --------------
    
    C:\PS>New-NaSnaplockLog -Basename priv_delete
    
    
    Create a new priv_delete log file.  The active priv_delete log file is 
    closed to futher updates and archived.
    
    
    
    LogVolume                LogBasename        LogFilename                     
                     FileSize
    ---------                -----------        -----------                     
                     --------
    snaplock_vol2            priv_delete        
    priv_delete.20110614_151302_GMT-present               259
    
    
    
    --------------  Example 2 --------------
    
    C:\PS>Get-NaSnaplockLog | New-NaSnaplockLog
    
    
    Close all currently active WORM log files and create new log files.
    
    
    
    LogVolume                LogBasename        LogFilename                     
                     FileSize
    ---------                -----------        -----------                     
                     --------
    snaplock_vol2            system_log         
    system_log.20110614_151519_GMT-present                257
    snaplock_vol2            priv_delete        
    priv_delete.20110614_151520_GMT-present               259
