Cmdlet: Set-NaCifsShareAcl


NAME
    Set-NaCifsShareAcl
    
SYNOPSIS
    Sets the access control list entry for the given CIFS share.
    
SYNTAX
    Set-NaCifsShareAcl [-Share] <String> [-User] <String> [-AccessRights] 
    <String> [-Controller <NaController>] [<CommonParameters>]
    
    Set-NaCifsShareAcl [-Share] <String> [-UnixGroup] <String> [-AccessRights] 
    <String> [-Controller <NaController>] [<CommonParameters>]
    
    Set-NaCifsShareAcl [-Share] <String> -Acl <CifsShareAclInfo> [-Controller 
    <NaController>] [<CommonParameters>]
    
    
DESCRIPTION
    Sets the access control list entry for the given CIFS share for a specific 
    user, group, or unix group.  This API is equivalent to "cifs access <share> 
    [-g] <user|group> <rights>" CLI.
    

PARAMETERS
    -Share <String>
        CIFS share name.  It specifies name of the share for which the access 
        rights have to be set.  This is a case insensitive field.  The maximum 
        size of the share name is 256 characters.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -User <String>
        Name of the user.  This API sets the access rights for the specified 
        user.  This is a case sensitive field.  If the specified user does not 
        exist, this API fails with the reason "Unknown user: <username>".
        
        Required?                    true
        Position?                    2
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -AccessRights <String>
        Access rights to be set to the above share and user.  The format of the 
        rights can be Unix-style combinations of r, w, x; or NT-style "No 
        Access", "Read", "Change", and "Full Control".
        
        Required?                    true
        Position?                    3
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -UnixGroup <String>
        Name of the unix group.  This field specifies the unix group for which 
        the access rights are to be set.  This is a case sensitive field.  If 
        the specified group name does not exist, this API fails with the reason 
        "Unknown Unix group : <groupname>".
        
        Required?                    true
        Position?                    2
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Acl <CifsShareAclInfo>
        An ACL object as returned by Get-NaCifsShareAcl.  This parameter set 
        simplifies copying ACLs among CIFS shares.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue)
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    DataONTAP.Types.Cifs.CifsShareInfo
        
        
    
     
    
NOTES
    
    
        Category: cifs
        API: cifs-share-ace-set
    
    --------------  Example 1 --------------
    
    C:\PS>Set-NaCifsShareAcl HOME root -AccessRights rw | Get-NaCifsShareAcl
    
    
    Add read/write rights to the CIFS share 'HOME' for the root user.
    
    
    
    ShareName UserAclInfo
    --------- -----------
    HOME      {everyone, root}
    
    
    
    --------------  Example 2 --------------
    
    C:\PS>Get-NaCifsShareAcl share1 | Set-NaCifsShareAcl share2 | ft -AutoSize
    
    
    Copy all ACL entries from CIFS share 'share1' to CIFS share 'share2'.
    
    
    
    MountPoint ShareName Description
    ---------- --------- -----------
    /vol/vol2  share2
