Cmdlet: Get-NaCifsStatistics


NAME
    Get-NaCifsStatistics
    
SYNOPSIS
    Display CIFS client statistics.
    
SYNTAX
    Get-NaCifsStatistics [-SortGroup <String>] [-MaxClients <Int32>] 
    [-AverageType <String>] [-Controller <NaController>] [<CommonParameters>]
    
    
DESCRIPTION
    Display CIFS client statistics.  The read/write size statistics are 
    provided in bytes/second.
    

PARAMETERS
    -SortGroup <String>
        If specified, the client statistics are sorted by the value of the 
        string: "ops":  Sort by the number of operations per second of any 
        type. "reads": Sort by kbytes/sec of data in response to read requests. 
        "writes": Sort by kbytes/sec of data written to the filer. "ios":  Sort 
        by the combined total of reads plus writes for each client. 
        "suspicious": Sort by the number of "suspicious" events per second by 
        each client.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -MaxClients <Int32>
        Specifies the maximum number of top clients to display (the default is 
        20).
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -AverageType <String>
        Specifies how the client statistics are to be averaged for display: 
        "smooth": Use a smoothed average which is weighted towards recent 
        behavior but takes into account previous history of the client. "now":  
        Use a one-second sample taken immediately and no history is taken into 
        account. "total": Use the total count of each statistic divided by the 
        total time since sampling started. If the is-verbose option is also 
        set, the totals are given without dividing by the sample time.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216).
