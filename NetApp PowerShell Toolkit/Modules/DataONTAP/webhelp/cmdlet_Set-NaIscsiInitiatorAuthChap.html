Cmdlet: Set-NaIscsiInitiatorAuthChap


NAME
    Set-NaIscsiInitiatorAuthChap
    
SYNOPSIS
    Modify CHAP parameters of an existing per-initiator authentication record 
    whose authentication type equals CHAP.
    
SYNTAX
    Set-NaIscsiInitiatorAuthChap [-Initiator] <String> [-Username <String>] 
    [-Password <String>] [-InboundCredential <PSCredential>] [-OutboundUsername 
    <String>] [-OutboundPassword <String>] [-OutboundCredential <PSCredential>] 
    [-RemoveOutbound] [-Controller <NaController>] [<CommonParameters>]
    
    
DESCRIPTION
    Modify CHAP parameters of an existing per-initiator authentication record 
    whose authentication type equals CHAP.
    

PARAMETERS
    -Initiator <String>
        Name of initiator (i.e. iqn.1987-06.com.initvendor1:appsrv.sn.2346). 
        The per-initiator authentication record must have an authentication 
        type equal to CHAP.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Username <String>
        Inbound CHAP user name.  If Inbound CHAP parameters are specified they 
        will replace the existing Inbound CHAP parameters.  If they are not 
        specified, the existing Inbound CHAP parameters will continue to be 
        used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Password <String>
        Inbound CHAP user password.  If Inbound CHAP parameters are specified 
        they will replace the existing Inbound CHAP parameters.  If they are 
        not specified, the existing Inbound CHAP parameters will continue to be 
        used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -InboundCredential <PSCredential>
        PSCredential object containing inbound CHAP credentials.  The username 
        and password supplied in the credential object take precedence over the 
        username and password supplied by the -Username and -Password 
        parameters.  If Inbound CHAP parameters are specified they will replace 
        the existing Inbound CHAP parameters.  If they are not specified, the 
        existing Inbound CHAP parameters will continue to be used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -OutboundUsername <String>
        Outbound CHAP user name.  If Outbound CHAP parameters are specified 
        they will replace existing Outbound CHAP parameters.  If no Outbound 
        CHAP parameters were previously specified, then the specified Outbound 
        CHAP parameters will enable mutual CHAP authentication.  If no Outbound 
        CHAP parameters are specified and no Outbound CHAP parameters exist, 
        then one-way Inbound CHAP authentication will be continue to be used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -OutboundPassword <String>
        Outbound CHAP user password.  If Outbound CHAP parameters are specified 
        they will replace existing Outbound CHAP parameters.  If no Outbound 
        CHAP parameters were previously specified, then the specified Outbound 
        CHAP parameters will enable mutual CHAP authentication.  If no Outbound 
        CHAP parameters are specified and no Outbound CHAP parameters exist, 
        then one-way Inbound CHAP authentication will be continue to be used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -OutboundCredential <PSCredential>
        PSCredential object containing outbound CHAP credentials.  The username 
        and password supplied in the credential object take precedence over the 
        username and password supplied by the -OutboundUsername and 
        -OutboundPassword parameters.  If Outbound CHAP parameters are 
        specified they will replace existing Outbound CHAP parameters.  If no 
        Outbound CHAP parameters were previously specified, then the specified 
        Outbound CHAP parameters will enable mutual CHAP authentication.  If no 
        Outbound CHAP parameters are specified and no Outbound CHAP parameters 
        exist, then one-way Inbound CHAP authentication will be continue to be 
        used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -RemoveOutbound
        Switch parameter which indicates that mutual CHAP authentication is to 
        be converted to one-way CHAP authentication.  Outbound CHAP parameters 
        must not be specified when RemoveOutbound is TRUE.
        
        Required?                    false
        Position?                    named
        Default value                false
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    DataONTAP.Types.Iscsi.IscsiSecurityEntryInfo
        
        
    
     
    
NOTES
    
    
        Category: iscsi
        API: iscsi-initiator-modify-chap-params
    
    --------------  Example 1 --------------
    
    C:\PS>Set-NaIscsiInitiatorAuthChap 
    iqn.1991-05.com.microsoft:x3550rre7.rtprre.testdomain -Username backupUser1 
    -Password safe4ever
    
    
    Set the CHAP authentication for the specified iSCSI initiator.
    
    
    
    AuthType            Initiator           OutboundUserName    UserName
    --------            ---------           ----------------    --------
    CHAP                iqn.1991-05.com....                     backupUser1
