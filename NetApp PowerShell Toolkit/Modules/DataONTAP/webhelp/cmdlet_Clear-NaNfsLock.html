Cmdlet: Clear-NaNfsLock


NAME
    Clear-NaNfsLock
    
SYNOPSIS
    Reclaims NFS locks.
    
SYNTAX
    Clear-NaNfsLock [[-Hosts] <String[]>] [-Controller <NaController>] 
    [<CommonParameters>]
    
    Clear-NaNfsLock -Host <String> -PID <String> [-Controller <NaController>] 
    [<CommonParameters>]
    
    
DESCRIPTION
    Reclaims the NFS locks for the specified client hosts. If no hosts are 
    specified, then all the clients locks are removed and are notified about 
    lock recovery, as if an NFS server crash/reboot had happened. If any hosts 
    are specified, then only those client hosts locks are reclaimed.  If a host 
    and process ID are specified, only the locks for that process are removed.
    

PARAMETERS
    -Hosts <String[]>
        Hosts whose locks have to be reclaimed. If no hosts are specified, then 
        all the clients locks are reclaimed.
        
        Required?                    false
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Host <String>
        Specify Host to reclaim the locks on one specific process on that host.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -PID <String>
        Specify PID to reclaim the locks on one specific process.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false

        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see
