Cmdlet: Remove-NaNfsMonitor


NAME
    Remove-NaNfsMonitor
    
SYNOPSIS
    Stops monitoring the specified hosts for NFS lock recovery purposes.
    
SYNTAX
    Remove-NaNfsMonitor [-Hosts] <String[]> [-Controller <NaController>] 
    [<CommonParameters>]
    
    
DESCRIPTION
    Stops monitoring the specified hosts for NFS lock recovery purposes. The 
    specified hosts are removed from the list of clients that will be notified 
    of lock recovery in the event of an NFS server crash/reboot.
    

PARAMETERS
    -Hosts <String[]>
        An array of hosts that are to be unmonitored.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216).
