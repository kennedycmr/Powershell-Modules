Cmdlet: Set-NaNetVif


NAME
    Set-NaNetVif
    
SYNOPSIS
    Modify a VIF.
    
SYNTAX
    Set-NaNetVif [-Name] <String> [-Add <String[]>] [-Remove <String[]>] 
    [-Favor <String>] [-Nofavor <String>] [-ActiveOnly] [-Controller 
    <NaController>] [<CommonParameters>]
    
    
DESCRIPTION
    Modify a VIF.  Modifies active and persistent config.  This cmdlet can add 
    interfaces to a VIF, remove interfaces from a VIF, and set favored and 
    non-favored interfaces.
    

PARAMETERS
    -Name <String>
        The name of the VIF to modify.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Add <String[]>
        List of interfaces to add to this VIF.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Remove <String[]>
        List of interfaces to remove from this VIF.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Favor <String>
        Interface that is favored in a single-mode VIF.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Nofavor <String>
        Interface that is not favored in a single-mode VIF.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -ActiveOnly
        If specified, only the active config is modified.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    DataONTAP.Types.Net.IfgrpInfo
        
        
    
     
    
NOTES
    
    
        Category: net
        API: system-cli
        Privilege: security-priv-admin, cli-vif-add
    
    --------------  Example 1 --------------
    
    C:\PS>Set-NaNetVif testvif -Add e0c,e0d -Favor e0c -Nofavor e0d
    
    
    Add interfaces 'e0c' and 'e0d' to single-mode VIF 'testvif', setting 'e0c' 
    as the favored interface and 'e0d' as the non-favored interface.
    
    
    
    Favored       : e0c
    IfgrpPolicy   : single
    IfgrpType     : single
    InterfaceName : testvif
    Links         : {e0d, e0c}
    Nofavored     : e0d
