Cmdlet: Set-NaFcpAdapterMediaType


NAME
    Set-NaFcpAdapterMediaType
    
SYNOPSIS
    Sets the link type on the Fibre Channel target adapter.
    
SYNTAX
    Set-NaFcpAdapterMediaType [-Adapter] <String> [-MediaType] <String> 
    [-Controller <NaController>] [<CommonParameters>]
    
    
DESCRIPTION
    Sets the link type on the Fibre Channel target adapter. It can be 
    configured to establish a point-to-point link, a loop configuration or to 
    automatically sense whether the connection type is a point-to-point or loop 
    link. Setting the link to point-to-point in a loop configuration can 
    prevent the loop from coming up properly. If the adapter is online, it must 
    be brought offline and then online in order for a new mediatype to take 
    effect. This may temporarily disrupt fcp service on the target adapter.
    

PARAMETERS
    -Adapter <String>
        FC adapter to modify.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -MediaType <String>
        Media type to set.  Valid values are point-to-point ("ptp"), loop 
        configuration ("loop"), or automatic ("auto").
        
        Required?                    true
        Position?                    2
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216).
