Cmdlet: Set-NaFpolicyVolumeList


NAME
    Set-NaFpolicyVolumeList
    
SYNOPSIS
    Manipulate a list of volumes in an exclude or include set.
    
SYNTAX
    Set-NaFpolicyVolumeList [-Name] <String> [-ListType] <String> [[-Volumes] 
    <String[]>] [-Controller <NaController>] [<CommonParameters>]
    
    
DESCRIPTION
    Manipulate a list of volumes in an exclude or include set.  This limits the 
    set of volumes for which client requests trigger (include) or suppress 
    (exclude) fpolicy processing for the provided policy. The list provided 
    will replace the list currently in place, if any. Note that if a policy has 
    both an exclude list and an include list, the include list is ignored by 
    the filer.
    

PARAMETERS
    -Name <String>
        Name of the fpolicy to set the volume list for.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -ListType <String>
        Defines to which set (exclude or include) a list will be applied. 
        Possible values: "exclude", "include".
        
        Required?                    true
        Position?                    2
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Volumes <String[]>
        List of volume specifications (including wild cards). The volumes are 
        case insensitive. If no volume list is provided, then the list will be 
        reset to an empty list. Supported wild card values: "?" to match any 
        character and "*" to match any number of characters. Example 
        specifications: vol0 vol? users*
        
        Required?                    false
        Position?                    3
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    DataONTAP.Types.Fpolicy.PolicyInfo
        
        
    
     
    
NOTES
    
    
        Category: fpolicy
        API: fpolicy-volume-list-set
    
    --------------  Example 1 --------------
    
    C:\PS>Set-NaFpolicyVolumeList policy2 include vol0 tmp1 | 
    Get-NaFpolicyVolumeList
    
    
    Sets the include volume list for policy2 to {vol0, tmp1} then gets the 
    volume list for policy2.
    
    
    
    Name    : policy2
    Include : {vol0, tmp1}
    Exclude : {}
    
    
    
    --------------  Example 2 --------------
    
    C:\PS>Set-NaFpolicyVolumeList policy2 include | Get-NaFpolicyVolumeList
    
    
    Resets the include volume list for fpolicy fpolicy2 to an empty list then 
    gets the volume list for fpolicy fpolicy2.
    
    
    
    Name    : policy2
    Include : {}
    Exclude : {}
