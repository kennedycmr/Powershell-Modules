Cmdlet: Set-NaPriorityVolume


NAME
    Set-NaPriorityVolume
    
SYNOPSIS
    Set priority scheduling for a volume.
    
SYNTAX
    Set-NaPriorityVolume [-Volume] <String> [-Service <String>] [-Level 
    <Int64>] [-System <Int64>] [-UserReadLimit <Int64>] [-SystemReadLimit 
    <Int64>] [-NvlogLimit <Int64>] [-CachePolicy <String>] [-Controller 
    <NaController>] [<CommonParameters>]
    
    
DESCRIPTION
    Set priority scheduling for a volume.
    

PARAMETERS
    -Volume <String>
        The volume whose priority scheduling is to be set.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Service <String>
        Sets the priority service type for the volume. The possible values are 
        'on', 'off' or 'default'.  'off' temporarily disables specific priority 
        scheduling for the volume, but preserves the priority schedule so that 
        it can later be enabled.  'default' removes the specific priority 
        scheduling for the volume, which will revert to using the default 
        priority schedule.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Level <Int64>
        Sets the priority schedule level for the volume.  The possible values 
        are 1 through 10, where 1 means lowest relative priority, and 10 means 
        highest relative priority.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -System <Int64>
        Sets the relative priority system/user scheduling for the volume.  The 
        possible values are 1 through 10, where 1 means heavily favor user 
        operations and 10 means heavily favor system operations.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -UserReadLimit <Int64>
        The maximum number of user read operations outstanding before further 
        operations are queued.  A value of 0 means restore the system-defined 
        limit.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -SystemReadLimit <Int64>
        The maximum number of system read operations outstanding before further 
        operations are queued.  A value of 0 means restore the system-defined 
        limit.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -NvlogLimit <Int64>
        The maximum percentage of system NVLOG resources that may be consumed.  
        A value of 0 means restore the system-defined limit.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -CachePolicy <String>
        The buffer cache policy to use for the volume.  Possible values are:
        
        'keep' - Try to keep data in the buffer cache
        'reuse' - Reuse blocks, do not try to cache
        'default' - Apply the default policy
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    DataONTAP.Types.Volume.VolumeInfo
