Cmdlet: Invoke-NaClusterTakeover


NAME
    Invoke-NaClusterTakeover
    
SYNOPSIS
    Initiates a takeover of the partner.
    
SYNTAX
    Invoke-NaClusterTakeover [-Force] [-Controller <NaController>] 
    [<CommonParameters>]
    
    
DESCRIPTION
    Initiates a takeover of the partner. Automatic giveback is enabled. Control 
    will be returned to partner once it boots up.
    

PARAMETERS
    -Force
        This causes takeover to be immediately initiated.  The taken over node, 
        if up, does not get to shut things down in an orderly manner so the 
        takeover of the resources takes longer. This is the type of takeover 
        that is done during normal cluster operation when one of the nodes goes 
        away (dies).
        
        Required?                    false
        Position?                    named
        Default value                false
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216).
