Cmdlet: Switch-NaDiskOwner


NAME
    Switch-NaDiskOwner
    
SYNOPSIS
    Changes ownership on disks already belonging to an owner.
    
SYNTAX
    Switch-NaDiskOwner [-OldOwner <String>] [-OldOwnerID <UInt32>] [-NewOwner 
    <String>] [-NewOwnerID <UInt32>] [-Force] [-Controller <NaController>] 
    [<CommonParameters>]
    
    
DESCRIPTION
    Changes ownership on disks already belonging to an owner.
    

PARAMETERS
    -OldOwner <String>
        Name of old owner.  This form takes all disks currently belonging to 
        the specific old owner, and reassigns them to a new owner.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -OldOwnerID <UInt32>
        ID of old owner.  This form takes all disks currently belonging to the 
        specific old owner ID, and reassign them to a new owner.  Either 
        OldOwner or OldOwnerID must be specified, but not both.  Range : 
        [0..2^32-1].
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -NewOwner <String>
        Name of new owner.  This form will assign all the disks belonging to 
        the old owner to the specific owner.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -NewOwnerID <UInt32>
        ID of new owner.  This form will assign all the disks belonging to the 
        old owner to the specific owner ID. Either NewOwner or NewOwnerID (or 
        both) must be specified.  Range : [0..2^32-1].
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Force
        Force flag need to be set to 'true' if reassigning disks which are 
        owned by another Filer.  However, if that Filer is up and has put a 
        reservation on the disk, even the force option won't work.  In this 
        case reassign will need to be run on the Filer which owns the disks.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216).

NOTES
    
    
        Category: disk
        API: disk-sanown-reassign
    
    --------------  Example 1 --------------
    
    C:\PS>Switch-NaDiskOwner -OldOwnerID 0118045471 -NewOwnerID 0118044195
    
    
    Switches ownership of disks from one controller to another.  The 
    controllers are specified by their system IDs, as returned by 
    "Get-NaDiskOwnerFiler".
