Cmdlet: Disable-NaNfsExportFence


NAME
    Disable-NaNfsExportFence
    
SYNOPSIS
    Disables fencing to the given exports for the given entry.
    
SYNTAX
    Disable-NaNfsExportFence [-Persistent] [-RemoveLocks] -Hosts <String[]> 
    -Paths <String[]> [-Controller <NaController>] [<CommonParameters>]
    
    Disable-NaNfsExportFence [-Persistent] [-RemoveLocks] -AllPaths -Hosts 
    <String[]> [-Controller <NaController>] [<CommonParameters>]
    
    Disable-NaNfsExportFence [-Persistent] [-RemoveLocks] -AllPaths -AllHosts 
    [-Controller <NaController>] [<CommonParameters>]
    
    Disable-NaNfsExportFence [-Persistent] [-RemoveLocks] -AllHosts -Paths 
    <String[]> [-Controller <NaController>] [<CommonParameters>]
    
    
DESCRIPTION
    Disables fencing to the given exports for the given entry. This means that 
    the entry will have write permission to the exports. The rule changes take 
    effect immediately. Set the persistent option to true to save the rule in 
    the /etc/exports file and keep the option persistent upon loading or reboot.
    

PARAMETERS
    -Persistent
        Default value is false. If true, modifies the etc/exports file to 
        append the rule for a permanent change. (The new rule still takes 
        effect immediately.)  If false, only change the exports in memory.
        
        Required?                    false
        Position?                    named
        Default value                false
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -RemoveLocks
        Default value is false. Set to true to reclaim locks of the specified 
        Hosts.
        
        Required?                    false
        Position?                    named
        Default value                false
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Hosts <String[]>
        An array of hostnames which are to be unfenced.
        
        A hostname can be ONE of the following formats.
        
         * machine-name: Alphanumeric string based on DNS.
         * netgroup: Alphanumeric string describing a group of machine names
         * IP: An IPV4 address in dotted decimal format AAA.BBB.CCC.DDD
         * IPv6: An IPv6 address in RFC 2732 format [AAAA:BBBB::GGGG]
         * IPv4 subnet: "[network] subnet [netmask] netmask"
         * IPv4/v6-subnet: IP/numbits. The IP is a subnet number and the 
        numbits specifies the size of the subnet by the number of leading bits 
        of the netmask.
         * dns: A DNS domain. An Alphanumeric starting with a '.'
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Paths <String[]>
        An array of paths which are to be unfenced.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -AllPaths
        Default value is false. Set to true to unfence all rules. Paths must be 
        left empty if this option is true.
        
        Required?                    true
        Position?                    named
        Default value                false
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -AllHosts
        Default value is false. Set to true to unfence all hosts. Hosts must be 
        left empty if this option is true.
        
        Required?                    true
        Position?                    named
        Default value                false
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS

NOTES
    
    
        Category: nfs
        API: nfs-exportfs-fence-disable
    
    --------------  Example 1 --------------
    
    C:\PS>Disable-NaNfsExportFence -Persistent -AllPaths -AllHosts
    
    
    Unfence all exports.
    
    
    
    
    
    --------------  Example 2 --------------
    
    C:\PS>Disable-NaNfsExportFence -Persistent -Hosts 10.10.10.10 -AllPaths
    
    
    Unfence all exports from host 10.10.10.10.
    
    
    
    
    
    --------------  Example 3 --------------
    
    C:\PS>Disable-NaNfsExportFence -Persistent -Paths /vol/vol2 -AllHosts
    
    
    Unfence all hosts from path /vol/vol2.
    
    
    
    
    
    --------------  Example 4 --------------
    
    C:\PS>Disable-NaNfsExportFence -Persistent -Paths /vol/vol2 -Hosts 
    .mycorp.com
    
    
    Unfence all hosts at .mycorp.com from path /vol/vol2.
