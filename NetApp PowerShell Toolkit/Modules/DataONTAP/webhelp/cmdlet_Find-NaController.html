Cmdlet: Find-NaController


NAME
    Find-NaController
    
SYNOPSIS
    Discover Data ONTAP controllers.
    
SYNTAX
    Find-NaController [-Community <String>] [-Address] <String> [-Timeout 
    <Int32>] [-NoPing] [-All] [<CommonParameters>]
    
    Find-NaController [-Community <String>] [-Address] <String> [-Mask] 
    <String> [-Timeout <Int32>] [-NoPing] [-All] [<CommonParameters>]
    
    Find-NaController [-Community <String>] -StartAddress <String> -EndAddress 
    <String> [-Timeout <Int32>] [-NoPing] [-All] [<CommonParameters>]
    
    
DESCRIPTION
    Discover Data ONTAP controllers.  This cmdlet is used in various ways to 
    find controllers on the network.  It uses SNMP to perform the discovery, so 
    SNMP must be enabled on the controllers.  It does not rely on any SNMP 
    functionality in the Microsoft Windows host operating system.  This cmdlet 
    does support IPv6.
    
    To discover a single controller:
    
    Find-NaController -Address 10.61.167.60
    
    To discover controllers within a range of addresses:
    
    Find-NaController -StartAddress 10.61.167.1 -EndAddress 10.61.167.100
    
    To discover controllers on a subnet:
    
    Find-NaController -Address 10.61.167.1 -Mask 255.255.255.0
    
    or use CIDR notation:
    
    Find-NaController -Address 10.61.167.0/24
    

PARAMETERS
    -Community <String>
        If specified, this is the community string to use during SNMP 
        discovery.  The default value is 'public'.
        
        Required?                    false
        Position?                    named
        Default value                public
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Address <String>
        Specify to discover a single address, or to specify any address within 
        a subnet when using the -Mask parameter, or to specify a subnet in CIDR 
        format.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Timeout <Int32>
        Timeout value for discovery, specified in milliseconds.  Default is 
        5000.
        
        Required?                    false
        Position?                    named
        Default value                5000
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -NoPing
        For efficiency, the discovery engine first attempts to ping a 
        controller before sending any SNMP commands.  But if ICMP traffic is 
        blocked within a network, specify this option to forego pinging and 
        proceed to the SNMP query stage.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -All
        Specify to return all SNMP devices, not just Data ONTAP controllers.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Mask <String>
        Specify in dotted quad notation when specifying a subnet to query.
        
        Required?                    true
        Position?                    2
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -StartAddress <String>
        The starting address within an address range to query.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -EndAddress <String>
        The ending address within an address range to query.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216).
