Cmdlet: Read-NaFile


NAME
    Read-NaFile
    
SYNOPSIS
    Read data from a named file.
    
SYNTAX
    Read-NaFile [-Path] <String> [-Raw] [-Controller <NaController>] 
    [<CommonParameters>]
    
    Read-NaFile [-Path] <String> [-Raw] [-Controller <NaController>] -Length 
    <Int64> -Offset <Int64> [<CommonParameters>]
    
    
DESCRIPTION
    Read data from a named file.
    
    Note that this API is provided primarily for administrative purposes.  It 
    is not intended for high-performance I/O.
    

PARAMETERS
    -Path <String>
        Path of the file, i.e. /vol/<volumename>/<filename>.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Raw
        Specify to get the file data as a byte array.  This option is useful 
        for non-ASCII files, such as binary or unicode files.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Length <Int64>
        Number of bytes to read from the file.  If not specified, the entire 
        file is read.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Offset <Int64>
        Offset into file to start reading from.  If not specified, the file is 
        read from the beginning.
        
        Required?                    true
        Position?                    named
        Default value                0
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    System.String

        Category: file
        API: file-read-file
