Cmdlet: Invoke-NaSnmpWalk


NAME
    Invoke-NaSnmpWalk
    
SYNOPSIS
    This cmdlet walks the SNMP MIB starting from the given OID.
    
SYNTAX
    Invoke-NaSnmpWalk [-OID] <String> [-Raw] [-Controller <NaController>] 
    [<CommonParameters>]
    
    
DESCRIPTION
    This cmdlet walks the SNMP MIB starting from the given OID.  This is a 
    convenience method, not a high-performance SNMP query tool.
    

PARAMETERS
    -OID <String>
        Fully qualified object identifier of an SNMP object. Only numeric OID's 
        (ex: .1.3.6.1.4.1.789.1.1.1.0) are allowed.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Raw
        Specify Raw to get string values.  Otherwise, a hashtable is written to 
        the output for each OID.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
