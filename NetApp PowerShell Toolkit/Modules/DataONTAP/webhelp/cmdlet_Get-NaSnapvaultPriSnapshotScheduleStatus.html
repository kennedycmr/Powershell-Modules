Cmdlet: Get-NaSnapvaultPriSnapshotScheduleStatus


NAME
    Get-NaSnapvaultPriSnapshotScheduleStatus
    
SYNOPSIS
    Request the primary to return status for all configured snapshot schedules.
    
SYNTAX
    Get-NaSnapvaultPriSnapshotScheduleStatus [[-VolumeName] <String>] 
    [-Controller <NaController>] [<CommonParameters>]
    
    
DESCRIPTION
    Request the primary to return status for all configured snapshot schedules. 
     If a specific volume is provided as input, this API will return only the 
    status for schedules within that volume.  When snapvault primary and 
    secondary are licensed on the same filer, output of this API is identical 
    to the output of Get-NaSnapvaultSecSnapshotScheduleStatus.
    

PARAMETERS
    -VolumeName <String>
        Primary volume for which snapshot schedule status is desired.  
        Wildcards are permitted.
        
        Required?                    false
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    DataONTAP.Types.Snapvault.SnapvaultSnapshotScheduleStatusInfo
        
        
    
     
    
NOTES
    
    
        Category: snapvault
