Cmdlet: Start-NaAggrVerify


NAME
    Start-NaAggrVerify
    
SYNOPSIS
    Start RAID mirror verification on the named aggregate.
    
SYNTAX
    Start-NaAggrVerify [[-Name] <String>] [-Fix] [-FixPlex <Int64>] 
    [-Controller <NaController>] [<CommonParameters>]
    
    
DESCRIPTION
    Start RAID mirror verification on the named aggregate. Verification 
    compares the data in both plexes of a mirrored aggregate.  In the default 
    case, any blocks that differ are logged and no changes are made. The 
    fix-plex option is used to fix any mismatches. It specifies which plex to 
    fix. 
    
    If no name is given, then mirror verification is started on all online 
    aggregates.  Use the "aggr-verify-list-info" API to check mirror 
    verification status. If the fix-plex option is used, then a name must be 
    specified.
    

PARAMETERS
    -Name <String>
        Name of the mirrored aggregate to verify.
        
        Required?                    false
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Fix
        If provided, and if the value is "true", then fix any discrepancies 
        found rather than just logging them.  The default value is "false". If 
        Fix is "true", then the FixPlex option must also be specified. If Fix 
        is "false" and FixPlex is also specified, then the Fix option will be 
        ignored.
        
        Required?                    false
        Position?                    named
        Default value                false
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -FixPlex <Int64>
        If provided, this specifies the plex to fix in case the two plexes do 
        not match. The default is to log any discrepancies instead of fixing 
        them.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216).
