Cmdlet: Get-NaHostVolume


NAME
    Get-NaHostVolume
    
SYNOPSIS
    Get Windows volume information.
    
SYNTAX
    Get-NaHostVolume [[-Path] <String>] [<CommonParameters>]
    
    Get-NaHostVolume [-VdsId] <Guid> [<CommonParameters>]
    
    Get-NaHostVolume [-DiskIndex] <UInt32> [<CommonParameters>]
    
    
DESCRIPTION
    Get Windows volume information.
    
    This cmdlet must be run with administrative privileges.
    

PARAMETERS
    -Path <String>
        The mount point (i.e. G:, 
        \\?\Volume{12345678-abcd-1234-abcd-1234567890ab}\) or a path to any 
        file on the volume to process
        
        Required?                    false
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -VdsId <Guid>
        The Guid object which uniquely identifies the volume in VDS.  Note that 
        this is not the same as the volume guid path.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -DiskIndex <UInt32>
        A windows disk index/number
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    DataONTAP.PowerShell.SDK.Cmdlets.Toolkit.Windows.HostVolume
