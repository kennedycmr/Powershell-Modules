Cmdlet: Set-NaDiskOwner


NAME
    Set-NaDiskOwner
    
SYNOPSIS
    Changes ownership on a disk.
    
SYNTAX
    Set-NaDiskOwner [-Name] <String> [[-Owner] <String>] [-OwnerID <UInt32>] 
    [-Pool <UInt32>] [-Checksum <String>] [-Force] [-All] [-DiskCount <Int32>] 
    [-DiskType <String>] [-Controller <NaController>] [<CommonParameters>]
    
    Set-NaDiskOwner [-Name] <String> [[-Owner] <String>] [-OwnerID <UInt32>] 
    [-Pool <UInt32>] [-Checksum <String>] [-Force] [-Auto] [-All] [-DiskCount 
    <Int32>] [-DiskType <String>] [-Controller <NaController>] 
    [<CommonParameters>]
    
    
DESCRIPTION
    Changes ownership on a disk. The normal usage is when there is no current 
    owner or to assign a disk to a pool.
    

PARAMETERS
    -Name <String>
        Name of disk to assign. Wildcarding for disk string is supported.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Owner <String>
        Assign disk to specific owner.
        
        Required?                    false
        Position?                    2
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -OwnerID <UInt32>
        Assign disk to specific owner ID (NVRAM ID or serial number). Range : 
        [0..2^32-1].
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Pool <UInt32>
        Assign disk to specific pool, e.g. '0' or '1'.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Checksum <String>
        Assign checksum type to disk. Option may only be specified on a gFiler. 
        Possible values: 'block' or 'zoned'.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Force
        Force flag need to be set to 'true' if assigning disks which are 
        already owned a Filer.  However, if that Filer is up and has put a 
        reservation on the disk, even the force option won't work.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -All
        Assign all unowned visible disks to the specified node.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -DiskCount <Int32>
        Assign specified count of disks.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -DiskType <String>
        Assign specified type of disk or set of disks.
        
        Type of disk: ATA, BSAS, EATA, FCAL, FSAS, LUN, MSATA, SAS, SATA, SCSI, 
        SSD, BSSD, XATA, XSAS, or unknown.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Auto
        Auto-assign unowned disks which are on loops where only one filer owns 
        the disks and the pool information is the same. No other input 
        parameters are allowed when specifying this option.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    DataONTAP.Types.Disk.DiskSanownDetailInfo
