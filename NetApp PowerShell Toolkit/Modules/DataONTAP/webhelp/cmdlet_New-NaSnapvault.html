Cmdlet: New-NaSnapvault


NAME
    New-NaSnapvault
    
SYNOPSIS
    Configure a new snapvault relationship.
    
SYNTAX
    New-NaSnapvault [-SecondaryPath] <String> [-PrimarySystem] <String> 
    [-PrimaryPath] <String> [-MaxTries <Int32>] [-MaxTransferRate <Int64>] 
    [-AllowOpenFileBackup [<Boolean>]] [-IgnoreAccessTimeChange [<Boolean>]] 
    [-UseCompression <String>] [-NoLunCloneExpansion [<Boolean>]] [-Controller 
    <NaController>] [<CommonParameters>]
    
    
DESCRIPTION
    Request the secondary system to configure a new snapvault relationship with 
    the given primary and secondary systems and paths.  This API is equivalent 
    to the 'snapvault start' Data ONTAP command.  All the inputs provided with 
    this request will be stored in the configuration entry maintained by the 
    secondary system.  These values will be used as default settings for 
    further incremental update transfers for this relationship.  The 
    Set-NaSnapvault cmdlet can be used to change these configured settings.  A 
    successful configuration will automatically be followed by a baseline 
    transfer from the primary to the secondary.  The secondary path will be 
    created during the baseline transfer hence it is required that the 
    secondary path must not exist when issuing this request.  This request will 
    only begin the baseline transfer and return. The transfer will proceed 
    asynchronously and there is no guarantee that it will succeed. The 
    Get-NaSnapvaultSecStatus cmdlet should be used to check the status of the 
    transfer.
    

PARAMETERS
    -SecondaryPath <String>
        The secondary path that will be used as destination for this 
        relationship as well as baseline transfer.  The secondary path will be 
        created during the baseline transfer and hence it must not exist when 
        issuing this request.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -PrimarySystem <String>
        The primary system for this relationship as well as for the baseline 
        transfer.  This input will be used by the secondary system to establish 
        contact with the primary.  Therefore this input is expected to be a 
        hostname that the secondary can resolve.
        
        Required?                    true
        Position?                    2
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -PrimaryPath <String>
        The primary path that will be used as the source for this relationship 
        as well as for baseline transfer.
        
        Required?                    true
        Position?                    3
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -MaxTries <Int32>
        The maximum number of times a transfer will be tried.  Transfers that 
        are retried using this mechanism may be capable of restarting from 
        where the previous attempt failed.  If a transfer does not succeed even 
        after those many attempts, then the secondary will give up.  All the 
        data that was transferred during previous tries for this transfer will 
        be discarded.  The default value for this parameter is 2.  When set to 
        0, the relationship will become dormant.  In other words no transfers 
        will be allowed to this secondary path.  The maximum value accepted for 
        this input is 120.  Range:[0..120]
        
        Required?                    false
        Position?                    named
        Default value                2
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -MaxTransferRate <Int64>
        The maximum transfer rate in bytes per second that will be used for 
        this relationship as well as baseline transfer.  The default value for 
        this parameter will allow transfers to proceed as fast as possible.  
        Range:[1..2^31-2]
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -AllowOpenFileBackup [<Boolean>]
        Sets the back_up_open_files option in the snapvault configuration 
        entry.  This option is used to allow or disallow the inclusion of open 
        files on the primary system at the time of the transfer.  This option 
        is currently applicable only for OSSV relationships.  When set to 
        'false' the OSSV primary agents will exclude files that are open from 
        the transfer.  The default value for this parameter is 'true'.
        
        Required?                    false
        Position?                    named
        Default value                true
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -IgnoreAccessTimeChange [<Boolean>]
        Sets the ignore_atime option in snapvault configuration entry.  When 
        set to 'true', snapvault primary does not send files with only access 
        time changes during incremental transfers.  The default value is 'false'
        
        Required?                    false
        Position?                    named
        Default value                false
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -UseCompression <String>
        Specifies whether to compress the network data stream.  Possible values 
        are 'on', 'off', and 'default'.  When the value is set to 'on', stream 
        compression will be enabled.  When the value is set to 'off', stream 
        compression will be disabled.  When the value is set to 'default' then 
        the default value will be used.  When this option is not specified, the 
        default value will be used.  The default value for this option is the 
        value of global compression option.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -NoLunCloneExpansion [<Boolean>]
        This option dictates how a lun clone would be transferred from source 
        to destination. If this option is "false", a LUN clone would be 
        transferred as a LUN and if it is "true", it will be transferred as a 
        clone. By default the value of the option is "false".
        
        Required?                    false
        Position?                    named
        Default value                false
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false

        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216).
