Cmdlet: Add-NaNfsExport


NAME
    Add-NaNfsExport
    
SYNOPSIS
    Enables pathnames for mounting according to the rules specified.
    
SYNTAX
    Add-NaNfsExport [-Persistent] [-Path] <String> [-ActualPath <String>] 
    [-SecurityFlavors <String[]>] [-Anon <String>] [-ReadOnly <String[]>] 
    [-ReadWrite <String[]>] [-Root <String[]>] [-NoSuid] [-Controller 
    <NaController>] [<CommonParameters>]
    
    
DESCRIPTION
    Enables pathnames for mounting according to the rules specified. New rules 
    for the pathnames take effect immediately, ignoring previous rules for 
    specified pathnames. Set the Persistent option to save the rule in the 
    etc/exports file and keep the option persistent upon loading or reboot.
    

PARAMETERS
    -Persistent
        Default value is false. If specified, modifies the etc/exports file to 
        append the rule for a permanent change. (The new rule still takes 
        effect immediately.)
        
        Required?                    false
        Position?                    named
        Default value                false
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Path <String>
        Directory name or file to export.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -ActualPath <String>
        Pathname inside of the filer which is being exported. The default for 
        this is value in Path.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -SecurityFlavors <String[]>
        List of possible security flavors this rule supports. Default security 
        is "sys".
        
        Required?                    false
        Position?                    named
        Default value                sys
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Anon <String>
        All hosts with this userid or username have root access to this 
        directory.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -ReadOnly <String[]>
        An array of hostnames which only have read privileges for all the 
        security flavors found in the SecurityFlavors list.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -ReadWrite <String[]>
        An array of hostnames which have read and write privileges for all the 
        security flavors found in the SecurityFlavors list. Any hostname in 
        ReadOnly must not be in ReadWrite also. Specify "all-hosts" to enable 
        read/write access from anywhere.  By default, if no ReadOnly or 
        ReadWrite hosts are given, then ReadWrite contains a hostname of 
        "all-hosts".
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Root <String[]>
        Array of hostnames which have roots with ReadWrite or ReadOnly 
        privileges.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -NoSuid
        If specified, causes the server file system to silently ignore any 
        attempt to enable the setuid or setgid mode bits. Default value is 
        false.
        
        Required?                    false
        Position?                    named
        Default value                false
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    DataONTAP.Types.Nfs.AddedRules
        
        
    
     
    
NOTES
    
    
        Category: nfs
        API: nfs-exportfs-append-rules-2
    
    --------------  Example 1 --------------
    
    C:\PS>Add-NaNfsExport /vol/testvol -Persistent -ReadWrite all-hosts -NoSuid 
    -SecurityFlavors sys,krb5
    
    
    Export volume 'testvol' with read/write access from all hosts.
