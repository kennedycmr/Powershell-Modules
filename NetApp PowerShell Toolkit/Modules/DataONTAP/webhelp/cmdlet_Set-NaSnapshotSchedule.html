Cmdlet: Set-NaSnapshotSchedule


NAME
    Set-NaSnapshotSchedule
    
SYNOPSIS
    Set the snapshot schedule on a specified volume.
    
SYNTAX
    Set-NaSnapshotSchedule [-TargetName] <String> [-Weeks <Int32>] [-Days 
    <Int32>] [-Hours <Int32>] [-Minutes <Int32>] [-WhichHours <String>] 
    [-WhichMinutes <String>] [-Controller <NaController>] [<CommonParameters>]
    
    Set-NaSnapshotSchedule [-TargetName] <String> [-Schedule] 
    <SnapshotSchedule> [-Controller <NaController>] [<CommonParameters>]
    
    
DESCRIPTION
    Set the snapshot schedule on a specified volume. If number of snapshots 
    requested is greater than ONTAP allows, then ESNAPTOOMANY will be returned 
    with the maximum allow snapshots in the reason.
    

PARAMETERS
    -TargetName <String>
        Name of the volume name where the snapshots are located.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Weeks <Int32>
        Number of snapshots taken weekly to keep on line.  If not provided, the 
        number of weekly snapshots is left at the previous value.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Days <Int32>
        Number of snapshots taken daily to keep on line.  If not provided, the 
        number of daily snapshots is left at the previous value.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Hours <Int32>
        Number of snapshots taken hourly to keep on line.  If not provided, the 
        number of weekly snapshots is left at the previous value.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Minutes <Int32>
        Number of snapshots taken minutely to keep on line.  If not provided, 
        the number of minutely snapshots is left at the previous value.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -WhichHours <String>
        Comma-separated list of the hours at which the hourly snapshots are 
        created.  If hours is 0, WhichHours is ignored and cleared.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -WhichMinutes <String>
        Comma-separated list of the minutes at which the minutely snapshots are 
        created.  If minutes is 0, WhichMinutes is ignored and cleared.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Schedule <SnapshotSchedule>
        Schedule object, as returned by Get-NaSnapshotSchedule.
        
        Required?                    true
        Position?                    2
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216).

OUTPUTS
