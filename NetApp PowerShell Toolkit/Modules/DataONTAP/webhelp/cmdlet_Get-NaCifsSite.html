Cmdlet: Get-NaCifsSite


NAME
    Get-NaCifsSite
    
SYNOPSIS
    Gathers a list of joinable sites units from Active Directory.
    
SYNTAX
    Get-NaCifsSite [-Domain] <String> [-User] <String> [-Password] <String> 
    [-Controller <NaController>] [-DefaultSite] [<CommonParameters>]
    
    Get-NaCifsSite [-Credential <PSCredential>] [-Controller <NaController>] 
    [-DefaultSite] [<CommonParameters>]
    
    
DESCRIPTION
    Gathers a list of joinable sites from Active Directory for the login-user 
    specified.  For more information regarding Active Directory sites and their 
    use, please reference Microsoft's Active Directory documentation.
    

PARAMETERS
    -Domain <String>
        The name of the domain that the CIFS server will join.  Examples: 
        cifsdomain, cifs.domain.com
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -User <String>
        The name of a domain user that has the ability to add the CIFS server 
        to the domain given in Domain.  Examples: username (assumes Domain is 
        the user's domain), cifsdomain\username, cifs.domain.com\username
        
        Required?                    true
        Position?                    2
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Password <String>
        The password for User.
        
        Required?                    true
        Position?                    3
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -DefaultSite
        Specify to limit the results to the default site listed in Active 
        Directory.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Credential <PSCredential>
        Credentials of a domain user that has the ability to add the CIFS 
        server to the domain.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    DataONTAP.Types.Cifs.CifsSetupSite
