Cmdlet category: Nfs

Add-NaNfsExport
Enables pathnames for mounting according to the rules specified.
Add-NaNfsMonitor
Starts monitoring the specified hosts for NFS lock recovery purposes.
Clear-NaNfsExportCache
For the given path, or all paths, renew or flush the access cache.
Clear-NaNfsLock
Reclaims NFS locks.
Clear-NaNfsStatistics
Set all NFS statistics to zero.
Disable-NaNfs
Stops the NFS service.
Disable-NaNfsExportFence
Disables fencing to the given exports for the given entry.
Enable-NaNfs
Starts the NFS service.
Enable-NaNfsExportFence
Enables fencing to the given exports for the given entry.
Get-NaNfsExport
Returns the rules associated with exports.
Get-NaNfsExportStoragePath
For the given path, determine the actual storage path.
Get-NaNfsMonitor
Lists the hosts that are currently being monitored by the NFS status monitor.
Get-NaNfsSecurityFlavor
Returns a list of currently supported security flavors.
Get-NaNfsStatistics
Collects NFS statistics for a specified client.
Get-NaNfsTopClient
Returns a list of the top NFS clients, ordered by total NFS operations.
Remove-NaNfsExport
Removes the rules for a set of pathnames.
Remove-NaNfsMonitor
Stops monitoring the specified hosts for NFS lock recovery purposes.
Set-NaNfsExport
Functionally similar to Add-NaNfsExport.
Test-NaNfs
Checks whether the NFS service is running.
Test-NaNfsExportPermission
Returns true if the host IP has mount permissions for a specified path.
Update-NaNfsExport
Loads the etc/exports file into memory.
