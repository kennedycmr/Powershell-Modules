Cmdlet: Get-NaSystemLog


NAME
    Get-NaSystemLog
    
SYNOPSIS
    Get the messages log, auditlog, ems log, or snapmirror log file.
    
SYNTAX
    Get-NaSystemLog -MessageLog [-Source <String[]>] [-EventType <String[]>] 
    [-Severity <String[]>] [-StartTime <DateTime>] [-EndTime <DateTime>] 
    [[-Path] <String>] [-Content <String>] [-Follow] [-Controller 
    <NaController>] [<CommonParameters>]
    
    Get-NaSystemLog -EmsLog [-Source <String[]>] [-EventType <String[]>] 
    [-StartTime <DateTime>] [-EndTime <DateTime>] [[-Path] <String>] [-Process 
    <String[]>] [-Follow] [-Controller <NaController>] [<CommonParameters>]
    
    Get-NaSystemLog -SnapMirror [-EventType <String[]>] [-StartTime <DateTime>] 
    [-EndTime <DateTime>] [[-Path] <String>] [-Content <String>] [-Follow] 
    [-Controller <NaController>] [<CommonParameters>]
    
    Get-NaSystemLog -AuditLog [-Source <String[]>] [-Severity <String[]>] 
    [-StartTime <DateTime>] [-EndTime <DateTime>] [[-Path] <String>] [-Keyword 
    <String[]>] [-Target <String[]>] [-Content <String>] [-Follow] [-Controller 
    <NaController>] [<CommonParameters>]
    
    
DESCRIPTION
    Get the messages log, auditlog, ems log, or snapmirror log file.
    

PARAMETERS
    -MessageLog
        If specified, the messages log files in /etc will be parsed.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Source <String[]>
        Filter log entries by source.  Wildcards are accepted.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -EventType <String[]>
        Filter log entries by event type.  Wildcards are accepted.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Severity <String[]>
        Filter log entries by severity level.  Wildcards are accepted.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -StartTime <DateTime>
        Display log entries that occurred after this time.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -EndTime <DateTime>
        Display log entries that occurred before this time.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Path <String>
        Absolute path to a log file to parse.
        
        Required?                    false
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Content <String>
        Log file content to parse.  This parameter can be used to parse 
        Snapmirror, Message, or AuditLog text directly without requiring the 
        cmdlet to retrieve the log text from the controller.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue)
        Accept wildcard characters?  false
        
    -Follow
        If specified, the last 4KB of the active log file will be read, then 
        the cmdlet will wait for further updates.  The cmdlet can be stopped 
        using ctrl-C.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -EmsLog
        If specified, the ems log files will be parsed.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Process <String[]>
        Filter log entries by owning process.  Wildcards are accepted.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -SnapMirror
        If specified, the snapmirror logs will be parsed.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -AuditLog
        If specified, the auditlog files will be parsed.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Keyword <String[]>
        Filter log entries by keyword.  Wildcards are accepted.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Target <String[]>
        Filter log entries by target.  Wildcards are accepted.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216).
