Cmdlet: Get-NaFpolicySecondaryServer


NAME
    Get-NaFpolicySecondaryServer
    
SYNOPSIS
    Gets a list of secondary servers for this fpolicy.
    
SYNTAX
    Get-NaFpolicySecondaryServer [-Name] <String> [-Controller <NaController>] 
    [<CommonParameters>]
    
    
DESCRIPTION
    Gets a list of secondary servers for this fpolicy.  These servers will be 
    used if all primary servers are not available, thus increasing system 
    availabilty.
    

PARAMETERS
    -Name <String>
        The name of the fpolicy to get secondary servers for.  Wildcards are 
        supported.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    DataONTAP.Types.Fpolicy.FpolicySecondaryServerInfo
        
        
    
     
    
NOTES
    
    
        Category: fpolicy
        API: fpolicy-get-secondary-servers-info
    
    --------------  Example 1 --------------
    
    C:\PS>Get-NaFpolicySecondaryServer policy*
    
    
    Get a list of secondary servers for all fpolicies whose names start with 
    "policy".
    
    
    
    Name        : policy1
    IpAddresses : {10.23.45.67, 10.61.52.43}
    
    Name        : policy2
    IpAddresses : {10.56.78.9}
