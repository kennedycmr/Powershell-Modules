Cmdlet: New-NaHostVolume


NAME
    New-NaHostVolume
    
SYNOPSIS
    Create a new Windows volume on an existing disk.
    
SYNTAX
    New-NaHostVolume [-DiskIndex] <UInt32> [[-MountPoint] <String>] [-NoMount] 
    [-Raw] [-Label <String>] [-Size <String>] [-AllocationUnitSize <Int32>] 
    [-Reclaim] [-UnmapSize <String>] [<CommonParameters>]
    
    
DESCRIPTION
    Create a new Windows volume on an existing disk.  This will automatically 
    initialize an unitialized/offline disk and optionally format the new volume 
    (by default)
    
    This cmdlet must be run with administrative privileges.
    

PARAMETERS
    -DiskIndex <UInt32>
        A windows disk index/number
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -MountPoint <String>
        A drive letter (i.e. 'G') or the path to an empty folder to mount the 
        new volume
        
        Required?                    false
        Position?                    2
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -NoMount
        Do not automatically assign a drive letter
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Raw
        Do not format the new volume, leave it as a raw partition
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Label <String>
        The label to assign the new file system
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Size <String>
        The size in bytes of the new volume
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -AllocationUnitSize <Int32>
        The allocation unit (cluster) size for the volume.  Valid values are 
        4KB, 8KB, 16KB, 32KB, and 64KB
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Reclaim
        Free any blocks which formerly occupied the space used by the new volume
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -UnmapSize <String>
        
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216).
