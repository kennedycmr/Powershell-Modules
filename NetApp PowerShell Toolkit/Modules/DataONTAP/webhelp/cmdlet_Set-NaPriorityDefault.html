Cmdlet: Set-NaPriorityDefault


NAME
    Set-NaPriorityDefault
    
SYNOPSIS
    Set priority options for default scheduling.
    
SYNTAX
    Set-NaPriorityDefault [-Level <Int64>] [-System <Int64>] [-UserReadLimit 
    <Int64>] [-SystemReadLimit <Int64>] [-NvlogLimit <Int64>] [-Controller 
    <NaController>] [<CommonParameters>]
    
    
DESCRIPTION
    Set priority options for default scheduling.  Default scheduling is applied 
    to all volumes that do not have a specific scheduling policy defined.
    

PARAMETERS
    -Level <Int64>
        Sets the priority schedule level for the volume.  The possible values 
        are 1 through 10, where 1 means lowest relative priority, and 10 means 
        highest relative priority.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -System <Int64>
        Sets the relative priority system/user scheduling for the volume.  The 
        possible values are 1 through 10, where 1 means heavily favor user 
        operations and 10 means heavily favor system operations.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -UserReadLimit <Int64>
        The maximum number of user read operations outstanding before further 
        operations are queued.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -SystemReadLimit <Int64>
        The maximum number of system read operations outstanding before further 
        operations are queued.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -NvlogLimit <Int64>
        The maximum percentage of system NVLOG resources that may be consumed 
        during a consistency point.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    
        
        
    
     
    
NOTES
    
    
        Category: priority
        API: priority-set-default
    
    --------------  Example 1 --------------
    
    C:\PS>Set-NaPriorityDefault -Level 6 -System 4
    
    
    Set the default volume priority settings to favor a volume more than most, 
    and to prioritize user I/O to a volume higher than system I/O.
