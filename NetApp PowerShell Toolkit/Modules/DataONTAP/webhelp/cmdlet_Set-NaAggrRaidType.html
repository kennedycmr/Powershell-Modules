Cmdlet: Set-NaAggrRaidType


NAME
    Set-NaAggrRaidType
    
SYNOPSIS
    
    
SYNTAX
    Set-NaAggrRaidType [-Name] <String> [-RaidType] <String> [-DiskType 
    <String>] [-Controller <NaController>] [<CommonParameters>]
    
    
DESCRIPTION
    Modify the RAID type for the given aggregate to the specified RaidType. 
    This cmdlet can also selectively change the RAID type of specific raid 
    groups in the aggregate based on the specified DiskType. The change remains 
    effective even if the filer is rebooted.  This cmdlet does not support 
    modifying options of a striped aggregate.
    

PARAMETERS
    -Name <String>
        Name or UUID of the aggregate whose option is to be set.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -RaidType <String>
        The type of RAID group used for this aggregate.P ossible values: 
        "raid4","raid_dp"
        
        The "raid4" setting  provides one parity disk per RAID group, while 
        "raid_dp" provides two.  Changing this option immediately changes the 
        RAID group type for the RAID groups in the aggregate.  When upgrading 
        RAID groups from "raid4" to "raid_dp", each RAID group begins a 
        reconstruction onto a spare disk allocated for the second "dparity" 
        parity disk.
        
        Required?                    true
        Position?                    2
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -DiskType <String>
        Type of disks to use : ATA, BSAS, FCAL, FSAS, LUN, SAS, SATA, or SSD.  
        This option is required if we want to change the raid type of raid 
        groups created out of the specified disks only.  Otherwise, if this is 
        not specified then the raid type for all raid groups in the aggregate 
        would change.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    DataONTAP.Types.Aggr.AggrInfo
        
        
    
     
    
NOTES
    
    
        Category: aggr
        API: aggr-modify-raid-type
    
    --------------  Example 1 --------------
    
    C:\PS>Set-NaAggrRaidType -Name aggr1 -RaidType raid_dp
    
    
    Change aggr1 to raid_dp.
