Cmdlet: Set-NaIscsiInitiatorDefaultAuth


NAME
    Set-NaIscsiInitiatorDefaultAuth
    
SYNOPSIS
    Configure the default iSCSI authentication method.
    
SYNTAX
    Set-NaIscsiInitiatorDefaultAuth [-None] [-Controller <NaController>] 
    [<CommonParameters>]
    
    Set-NaIscsiInitiatorDefaultAuth -Chap [-Username <String>] [-Password 
    <String>] [-InboundCredential <PSCredential>] [-OutboundUsername <String>] 
    [-OutboundPassword <String>] [-OutboundCredential <PSCredential>] 
    [-Controller <NaController>] [<CommonParameters>]
    
    Set-NaIscsiInitiatorDefaultAuth -Deny [-Controller <NaController>] 
    [<CommonParameters>]
    
    
DESCRIPTION
    Configure the default iSCSI authentication method.  If an initiator is not 
    configured with a specific authentication method using 
    Add-NaIscsiInitiatorAuth the default authentication method will be applied 
    to it.
    

PARAMETERS
    -None
        Specify this switch parameter for authentication type 'none'.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Chap
        Specify this switch parameter for authentication type 'CHAP'.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Username <String>
        Inbound CHAP user name.  For the CHAP authentication type, the 
        -Username and -Password or -InboundCredential parameters must be 
        specified.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Password <String>
        Inbound CHAP user password.  For the CHAP authentication type, the 
        -Username and -Password or -InboundCredential parameters must be 
        specified.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -InboundCredential <PSCredential>
        PSCredential object containing inbound CHAP credentials.  The username 
        and password supplied in the credential object take precedence over the 
        username and password supplied by the -Username and -Password 
        parameters.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -OutboundUsername <String>
        Outbound CHAP user name.  Outbound authentication is optional. If 
        Outbound authentication is not specified, then the initiator can only 
        do inbound traffic.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -OutboundPassword <String>
        Outbound CHAP user password.  Outbound authentication is optional. If 
        Outbound authentication is not specified, then the initiator can only 
        do inbound traffic.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -OutboundCredential <PSCredential>
        PSCredential object containing outbound CHAP credentials.  The username 
        and password supplied in the credential object take precedence over the 
        username and password supplied by the -OutboundUsername and 
        -OutboundPassword parameters.  Outbound authentication is optional. If 
        Outbound authentication is not specified, then the initiator can only 
        do inbound traffic.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Deny
        Specify this switch parameter for authentication type 'deny'.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
