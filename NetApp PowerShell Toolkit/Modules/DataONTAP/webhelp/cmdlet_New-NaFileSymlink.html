Cmdlet: New-NaFileSymlink


NAME
    New-NaFileSymlink
    
SYNOPSIS
    Create a symlink.
    
SYNTAX
    New-NaFileSymlink [-Symlink] <String> [-Path] <String> [-Controller 
    <NaController>] [<CommonParameters>]
    
    
DESCRIPTION
    Create a symlink.
    

PARAMETERS
    -Symlink <String>
        Value of the symlink, i.e. what the symlink points to.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Path <String>
        Path of the symlink, i.e. /vol/<volumename>/<symlinkname>.
        
        Required?                    true
        Position?                    2
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    DataONTAP.Types.File.FileInfo
        
        
    
     
    
NOTES
    
    
        Category: file
        API: file-create-symlink

    C:\PS>New-NaFileSymlink /vol/vol0/etc/hosts /vol/vol0/etc/symhosts
