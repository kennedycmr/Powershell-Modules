Cmdlet: Start-NaNdmpCopy


NAME
    Start-NaNdmpCopy
    
SYNOPSIS
    Copy content between controllers using NDMP.
    
SYNTAX
    Start-NaNdmpCopy [-SrcController] <IPAddress> [-SrcPath] <String> 
    [[-DstController] <IPAddress>] [-DstPath] <String> [-SrcCredential 
    <PSCredential>] [-DstCredential <PSCredential>] [-SrcAuthType 
    <NdmpMessageAuthType>] [-DstAuthType <NdmpMessageAuthType>] [-SrcPort 
    <Int32>] [-DstPort <Int32>] [-Version <Int16>] [-Exclude <String>] [-Level 
    <Int32>] [-UseIpv6] [-Transient] [<CommonParameters>]
    
    Start-NaNdmpCopy [-SrcController] <IPAddress> [-SrcPath] <String> 
    [[-DstController] <IPAddress>] [-DstPath] <String> [-SrcCredential 
    <PSCredential>] [-DstCredential <PSCredential>] [-SrcAuthType 
    <NdmpMessageAuthType>] [-DstAuthType <NdmpMessageAuthType>] [-SrcPort 
    <Int32>] [-DstPort <Int32>] [-Version <Int16>] [-Exclude <String>] 
    [-UseIpv6] -IncrementalForever [-Transient] [<CommonParameters>]
    
    
DESCRIPTION
    Copy content between controllers using NDMP.  Unlike Invoke-NaNdmpCopy, 
    Start-NaNdmpCopy runs asynchronously.  Once the operation has been started 
    the cmdlet will return.  Use Get-NaNdmpCopy to retrieve the status of a 
    running NDMP copy operation.  The PowerShell session must remain open for 
    the duration of the NDMP copy operation or the operation will fail.
    

PARAMETERS
    -SrcController <IPAddress>
        IP address of the source controller.  For controllers operating in 
        Cluster-Mode, this must be an IP of the node hosting the volume 
        containing the content to be copied.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -SrcPath <String>
        Absolute path of the source directory to transfer.  For a controller 
        operating in 7-mode, must start with /vol/<volname>.  For a controller 
        operating in Cluster-Mode, must start with /<vserver>/<volname>.
        
        Required?                    true
        Position?                    2
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -DstController <IPAddress>
        IP address of the destination controller.  For controllers operating in 
        Cluster-Mode, this must be an IP of the node hosting the destination 
        volume.
        
        If not specified, the SrcController is used as the destination.
        
        Required?                    false
        Position?                    3
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -DstPath <String>
        Absolute path of the destination directory.  If it does not exist, it 
        will be created.  For a controller operating in 7-mode, must start with 
        /vol/<volname>.  For a controller operating in Cluster-Mode, must start 
        with /<vserver>/<volname>.
        
        Required?                    true
        Position?                    4
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -SrcCredential <PSCredential>
        Credential to use to authenticate connection to source controller.  For 
        controllers operating in Cluster-Mode, credentials are configured using 
        the CLI command "system services ndmp password".  If not supplied and 
        the SrcAuthType is not None, the credentials cache will be consulted.  
        If no credentials are found, the cmdlet will fail.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -DstCredential <PSCredential>
        Credential to use to authenticate connection to destination controller. 
         For controllers operating in Cluster-Mode, credentials are configured 
        using the CLI command "system services ndmp password".  If not supplied 
        and the DstAuthType is not None, the credentials cache will be 
        consulted.  If no credentials are found, the cmdlet will fail.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -SrcAuthType <NdmpMessageAuthType>
        Authentication type to use when authenticating connection to source 
        controller.  Possible values: text, md5, none.  Default is none.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -DstAuthType <NdmpMessageAuthType>
        Authentication type to use when authenticating connection to 
        destination controller.  Possible values: text, md5, none.  Default is 
        none.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -SrcPort <Int32>
        Port to connect to NDMP service on the source controller.  Default is 
        10000.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -DstPort <Int32>
        Port to connect to NDMP service on the destination controller.  Default 
        is 10000.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Version <Int16>
        NDMP version to use when communicating with source and destination 
        controllers.  Must be at least 3, Default is 4.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Exclude <String>
        Exclude string.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Level <Int32>
        The incremental level to be used for the transfer is restricted to 0, 1 
        or 2 only. The default level used is 0.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -UseIpv6

        Required?                    false

        Required?                    false
        Position?                    named
        Default value

        Accept wildcard characters?  false
        
    -IncrementalForever

        Required?                    true
        Position?                    named
