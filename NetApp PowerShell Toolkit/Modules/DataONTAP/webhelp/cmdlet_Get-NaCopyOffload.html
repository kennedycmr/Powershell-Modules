Cmdlet: Get-NaCopyOffload


NAME
    Get-NaCopyOffload
    
SYNOPSIS
    Provides status on an on-going copyoffload-copy-file request. The response 
    will indicate the state of the on-going operation and the number of bytes 
    copied thus far.
    
SYNTAX
    Get-NaCopyOffload [[-CopyId] <String>] [-Controller <NaController>] 
    [<CommonParameters>]
    
    
DESCRIPTION
    Provide status on an on-going copyoffload-copy-file request. The response 
    will indicate the state of the on-going operation and the number of bytes 
    copied thus far.
    

PARAMETERS
    -CopyId <String>
        Unique identifier for an on-going copy operation.
        
        Required?                    false
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    DataONTAP.Types.Copyoffload.CopyoffloadStatusInfo
        
        
    
     
    
NOTES
    
    
        Category: copyoffload
        API: copyoffload-copy-status
