Cmdlet: Add-NaCifsShare


NAME
    Add-NaCifsShare
    
SYNOPSIS
    Creates a new CIFS share rooted at the specified path.
    
SYNTAX
    Add-NaCifsShare [-Share] <String> [-Path] <String> [-Comment <String>] 
    [-MaxUsers <Int32>] [-Group <String>] [-Widelink [<Boolean>]] 
    [-SymlinkStrictSecurity [<Boolean>]] [-VirusScan [<Boolean>]] 
    [-VirusScanRead [<Boolean>]] [-Browse [<Boolean>]] [-Umask <Int32>] 
    [-DirectoryUmask <Int32>] [-FileUmask <Int32>] [-Caching <String>] 
    [-AccessBasedEnum] [-NamespaceChaching] [-Controller <NaController>] 
    [<CommonParameters>]
    
    
DESCRIPTION
    Creates a new CIFS share rooted at the specified path.
    

PARAMETERS
    -Share <String>
        Name of the share to be added. The name cannot exceed 12 characters for 
        DOS-style shares and 256 characters for others.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Path <String>
        Full path name of the directory on the filer corresponding to the root 
        of the new share.
        
        Required?                    true
        Position?                    2
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Comment <String>
        If specified, gives description of the new share. CIFS clients see this 
        description when browsing the filer's shares. If not specified, the 
        description is blank.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -MaxUsers <Int32>
        If specified, gives the maximum number of simultaneous connections to 
        the new share. It must be a positive number. If not specified, the 
        filer does not impose a limit on the number of connections to the share.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Group <String>
        If specified, provides name of the group to which files to be created 
        in the share belong. The groupname is the name of a group in the UNIX 
        group database. If it is an empty string or else not specified, then 
        files to be created in the share do not belong to a particular UNIX 
        group. That is, each file belongs to the same group as the owner of the 
        file.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Widelink [<Boolean>]
        If true, allows clients to follow absolute symbolic links outside of 
        this share, subject to NT security. This feature requires an entry in 
        the /etc/symlink.translations file and it requires that the client 
        supports  Microsoft's Distributed File System (DFS). If false or not 
        specified, widelinks in the share are disabled.
        
        Required?                    false
        Position?                    named
        Default value                false
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -SymlinkStrictSecurity [<Boolean>]
        If true or not specified, strict symlink security is enabled. If false, 
        allows clients to follow symbolic links to destinations on this filer 
        but outside of the current share. It is not checked if the client is 
        authenticated to the symbolic link's destination.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -VirusScan [<Boolean>]
        If true or not specified, virus scan is done when clients open files on 
        this share, else no virus scan is done.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -VirusScanRead [<Boolean>]
        If true or not specified, virus scan is done when clients open files on 
        this share for read access, else no virus scan is done for read access 
        on this share.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Browse [<Boolean>]
        If true or not specified, shares are visible to browsing clients and 
        can be enumerated (e.g. "net view", SrvMgr, AD).
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Umask <Int32>
        If specified, sets file mode creation mask for a share in qtrees with 
        Unix or mixed security styles. The mask restricts the initial 
        permissions setting of a newly created file or directory. If not 
        specified, the file mode creation mask of the share is 0. This field is 
        ignored when both DirectoryUmask and FileUmask are present.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -DirectoryUmask <Int32>
        If specified, sets file mode creation mask for a share in qtrees with 
        Unix or mixed security styles. The mask restricts the initial 
        permissions setting of a newly created directory. For directories, this 
        mask overrides one set with "Umask".
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -FileUmask <Int32>
        If specified, sets file mode creation mask for a share in qtrees with 
        Unix or mixed security styles. The mask restricts the initial 
        permissions setting of a newly created file. For files, this mask 
        overrides one set with "Umask".
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Caching <String>
        If specified, the following is done based on the value of the string: 
        "no_caching": disallow Windows clients from caching any files on this 
        share. This is the default value. "auto_document_caching": allow 
        Windows clients to cache user documents on this share. 
        "auto_program_caching": allow Windows clients to cache programs on this 
        share. The actual caching behavior depends on the Windows client.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -AccessBasedEnum
        If specified, Access Based Enumeration (ABE) is enabled, else it is 
        disabled. ABE filtered shared folders are visible to a user based on 
        that individual user's access rights, preventing the display of folders 
        or other shared resources that the user does not have rights to access.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -NamespaceChaching
        If specified, namespace caching is enabled on the share. If namespace 
        caching is enabled on a share, clients are allowed to cache the 
        directory enumeration results for better performance.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
