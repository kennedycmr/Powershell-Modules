Cmdlet category: Cifs

Add-NaCifsShare
Creates a new CIFS share rooted at the specified path.
Disable-NaCifs
Stops the CIFS service for all users or a particular workstation, if specified.
Enable-NaCifs
Starts the CIFS service.
Get-NaCifs
Display the CIFS configuration.
Get-NaCifsBranchCacheHashStatistics
Get the CIFS BranchCache statistics.
Get-NaCifsHomeDirectory
Return path to the user's CIFS home directory if it exists, or return the paths to all home directories.
Get-NaCifsNetBiosAlias
Return the current list of NetBIOS alias names for the controller.
Get-NaCifsOrganizationalUnit
Gathers a list of joinable organizational units from Active Directory.
Get-NaCifsSession
Gives information on current CIFS activity.
Get-NaCifsShare
Gives information about one or more shares.
Get-NaCifsShareAcl
Gives information about one or more CIFS shares' access control list (ACL).
Get-NaCifsSite
Gathers a list of joinable sites units from Active Directory.
Get-NaCifsStatistics
Display CIFS client statistics.
New-NaCifsGroupFile
Creates a basic /etc/group file.
New-NaCifsPasswordFile
Creates a basic /etc/passwd file including a root user with the specified password.
Remove-NaCifsShare
Deletes the specified CIFS share.
Remove-NaCifsShareAcl
Deletes an access control entry of the given share.
Set-NaCifs
Configures the filer's CIFS service.
Set-NaCifsBranchCacheKey
Sets the server secret for BranchCache.
Set-NaCifsHomeDirectory
Provides a list of CIFS home directory paths for the controller.
Set-NaCifsNetBiosAlias
Provides a list of NetBIOS alias names for the controller.
Set-NaCifsShare
Changes settings of one or more CIFS shares at any time, even if the shares are in use.
Set-NaCifsShareAcl
Sets the access control list entry for the given CIFS share.
Test-NaCifs
Returns the running state of the CIFS service.
Test-NaCifsName
Determines whether or not a particular CIFS server name is already in use on the network and a specified domain.
Test-NaCifsPasswordGroupFile
Determines whether or not the /etc/passwd and /etc/group files exist, which would be used in the event that the configured CIFS authentication method fails.
