Cmdlet: Set-NaSnapmirrorSyncSchedule


NAME
    Set-NaSnapmirrorSyncSchedule
    
SYNOPSIS
    Establishes a synchronous schedule.
    
SYNTAX
    Set-NaSnapmirrorSyncSchedule [-Destination] <String> [-Source] <String> 
    [[-VisibilityFrequency] <Int32>] [[-OpsThrottle] <String>] [-SemiSync] 
    [-IsCompressed [<Boolean>]] [-TcpWindowSize <Int32>] [-Controller 
    <NaController>] [<CommonParameters>]
    
    Set-NaSnapmirrorSyncSchedule [-Destination] <String> [-Source] <String> 
    [-Controller <NaController>] -Schedule <SnapmirrorSyncScheduleInfo> 
    [<CommonParameters>]
    
    
DESCRIPTION
    Establishes a synchronous schedule. Currently, the schedules are in: 
    /etc/snapmirror.conf.
    

PARAMETERS
    -Destination <String>
        The destination location of a schedule to set.  The destination 
        location is of the volume form: <filer>:<volume> or the qtree form: 
        <filer>:/vol/<volume>/<qtree>.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Source <String>
        The source location of a schedule to set.  The source location is of 
        the volume form: <filer>:<volume> or the qtree form: 
        <filer>:/vol/<volume>/<qtree>.
        
        Required?                    true
        Position?                    2
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -VisibilityFrequency <Int32>
        Controls how often the source snapspot will be visible on the 
        destination mirror. This input is used to control the value of 
        visibility_interval in the snapmirror.conf file. The units are in 
        seconds. A typical value to use for this input is 180.
        
        Required?                    false
        Position?                    3
        Default value                180
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -OpsThrottle <String>
        The number of outstanding operations allowed before blocking on the 
        source.  The format is a number follwed by the one of the following 
        units: "ops", "s", or "ms". If the specified value is less than 10s, 
        the mirror is configured to run in a fully synchronous mode. If the 
        specified value is greater than or equal to 10s, the mirror is 
        configured to run in semi-synchronous mode.
        
        Required?                    false
        Position?                    4
        Default value                
        Accept pipeline input?       true (ByPropertyName)
        Accept wildcard characters?  false
        
    -SemiSync
        This specifies that the mirror should be configured in semi-sync mode.  
        If not specified, the mirror will be configured to run in full 
        synchronous mode.  This parameter overrides the deprecated OpsThrottle 
        parameter.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -IsCompressed [<Boolean>]
        If true, SnapMirror will compress/decompress the data that is 
        transferred between the source and destination storage system. If 
        false, transferred data will not be compressed. Upon initial 
        configuration, default is false. On subsequent requests, the current 
        configured setting is retained if not provided.  This parameter can 
        only be used when a connection definition is used for the relationship 
        entry.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -TcpWindowSize <Int32>
        TCP window size in bytes.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Schedule <SnapmirrorSyncScheduleInfo>
        Alternate parameter set that accepts a schedule object of the type 
        returned by Get-NaSnapmirrorSyncSchedule.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       true (ByValue)
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
