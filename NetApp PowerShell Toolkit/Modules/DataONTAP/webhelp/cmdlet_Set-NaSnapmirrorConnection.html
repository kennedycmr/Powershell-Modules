Cmdlet: Set-NaSnapmirrorConnection


NAME
    Set-NaSnapmirrorConnection
    
SYNOPSIS
    Set up a connection.  Set-NaSnapmirrorConnection will add a new connection 
    or modify an existing one. This cmdlet must be executed on the destination 
    controller.  The connections are saved in /etc/snapmirror.conf.
    
SYNTAX
    Set-NaSnapmirrorConnection [-Name] <String> [[-Mode] <String>] 
    [-FirstSource] <String> [-FirstDestination] <String> [-Controller 
    <NaController>] [<CommonParameters>]
    
    Set-NaSnapmirrorConnection [-Name] <String> [[-Mode] <String>] 
    [-FirstSource] <String> [-FirstDestination] <String> [-SecondSource] 
    <String> [-SecondDestination] <String> [-Controller <NaController>] 
    [<CommonParameters>]
    
    
DESCRIPTION
    Set up a connection.  Set-NaSnapmirrorConnection will add a new connection 
    or modify an existing one. This cmdlet must be executed on the destination 
    controller.  The connections are saved in /etc/snapmirror.conf.
    

PARAMETERS
    -Name <String>
        Name of the connection to add or modify.  The name is in ASCII and must 
        begin with an alpha character.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Mode <String>
        Mode of the connection.  Possible mode values are "multi" or 
        "failover". If not specified, the default is "multi".
        
        Required?                    false
        Position?                    2
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -FirstSource <String>
        The connection's first source address.  In multi mode, the first 
        address pair provides a connection path; while in failover mode, the 
        first address pair provides the prefer connection path.
        
        Required?                    true
        Position?                    3
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -FirstDestination <String>
        The connection's first destination address.  In multi mode, the first 
        address pair provides a connection path; while in failover mode, the 
        first address pair provides the prefer connection path.
        
        Required?                    true
        Position?                    4
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -SecondSource <String>
        The connection's second source address.  In multi mode the second 
        address pair provides another connection path, while in failover mode, 
        the second address pair provides a connection path in case the first 
        path fails.
        
        Required?                    true
        Position?                    5
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -SecondDestination <String>
        The connection's second destination address.  In multi mode the second 
        address pair provides another connection path, while in failover mode, 
        the second address pair provides a connection path in case the first 
        path fails.
        
        Required?                    true
        Position?                    6
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    DataONTAP.Types.Snapmirror.ConnectionInfo
        
        
    
     
    
NOTES
    
    
        Category: snapmirror
        API: snapmirror-set-connection
    
    --------------  Example 1 --------------
    
    C:\PS>Set-NaSnapmirrorConnection -Name test-con -Mode multi -FirstSource 
    benson -FirstDestination dunn
    
    
    Set a connection named "test-con" with a source of benson and a destination 
    of dunn.
    
    
    
    Name               Mode       AddressPair1                     AddressPair2
    ----               ----       ------------                     ------------
    test-con           multi      benson, dunn
