Cmdlet: Remove-NaNfsExport


NAME
    Remove-NaNfsExport
    
SYNOPSIS
    Removes the rules for a set of pathnames.
    
SYNTAX
    Remove-NaNfsExport [-Paths] <String[]> [-Persistent] [-Controller 
    <NaController>] [<CommonParameters>]
    
    Remove-NaNfsExport [-AllPaths] [-Persistent] [-Controller <NaController>] 
    [<CommonParameters>]
    
    
DESCRIPTION
    Removes the rules for a set of pathnames. This returns an error if any of 
    the pathnames don't have a rule. Set the Persistent option to modify the 
    etc/exports file and keep this change persistent upon reboots.
    

PARAMETERS
    -Paths <String[]>
        Pathnames to be deleted from the exports table.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Persistent
        Default value is false.  Modify the etc/exports file to delete the 
        rules permanently.  CAUTION: If AllPaths and Persistent are both 
        specified, all exports are removed permanently.
        
        Required?                    false
        Position?                    named
        Default value                false
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -AllPaths
        Default value is false. Specify to delete all rules. Paths must be left 
        empty if this option is true.
        
        Required?                    false
        Position?                    named
        Default value                false
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    DataONTAP.Types.Nfs.DeletedRules
        
        
    
     
    
NOTES
    
    
        Category: nfs
        API: nfs-exportfs-delete-rules
    
    --------------  Example 1 --------------
    
    C:\PS>Remove-NaNfsExport -Persistent /vol/testvol
    
    
    Remove the export for volume 'testvol'.
    
    
    
    DeletedPathnames                                            
    UnexportedPathnames
    ----------------                                            
    -------------------
    {/vol/testvol}                                              {/vol/testvol}
