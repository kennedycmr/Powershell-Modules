Cmdlet: Suspend-NaAggrScrub


NAME
    Suspend-NaAggrScrub
    
SYNOPSIS
    Suspend parity scrubbing on the named aggregate, plex, or RAID group.
    
SYNTAX
    Suspend-NaAggrScrub [[-Name] <String>] [-Controller <NaController>] 
    [<CommonParameters>]
    
    
DESCRIPTION
    Suspend parity scrubbing on the named aggregate, plex, or RAID group.  If 
    no name is given, suspend scrubbing on all RAID groups currently being 
    scrubbed.
    

PARAMETERS
    -Name <String>
        Name of the aggregate, plex, or RAID group for which we are to suspend 
        parity scrubbing using the following format: 
        
        [/vol/]<aggrname>[<plexinfo>][<groupinfo>]
        
        Required?                    false
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    DataONTAP.Types.Volume.ScrubDetailInfo
        
        
    
     
    
NOTES
    
    
        Category: aggr
        API: aggr-scrub-suspend
