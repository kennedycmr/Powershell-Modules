Cmdlet: Invoke-NaVolMoveCutover


NAME
    Invoke-NaVolMoveCutover
    
SYNOPSIS
    Initiate manual cutover.  If cutover cannot be initiated or completed, the 
    API will return with an error. The move will pause and an EMS message will 
    be printed. The Get-NaVolMove cmdlet will show the state of the move as 
    move(paused). The user can resume or abort the move.
    
SYNTAX
    Invoke-NaVolMoveCutover [-Volume] <String> [[-CutoverWindow] <Int32>] 
    [-Controller <NaController>] [<CommonParameters>]
    
    
DESCRIPTION
    Initiate manual cutover.  If cutover cannot be initiated or completed, the 
    API will return with an error. The move will pause and an EMS message will 
    be printed. The Get-NaVolMove cmdlet will show the state of the move as 
    move(paused). The user can resume or abort the move.
    

PARAMETERS
    -Volume <String>
        Name of the volume that is being moved.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -CutoverWindow <Int32>
        Time interval to complete cutover in seconds.  Default is the value the 
        value given to Start-NaVolMove.
        
        Required?                    false
        Position?                    2
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216).
