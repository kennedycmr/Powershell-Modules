Cmdlet: Set-NaVfilerAddress


NAME
    Set-NaVfilerAddress
    
SYNOPSIS
    Manage vfiler addresses.
    
SYNTAX
    Set-NaVfilerAddress [-Name] <String> [-AddAddresses <String[]>] 
    [-RemoveAddresses <String[]>] [-IpBindingInfo <IpbindingInfo[]>] 
    [-PreserveEtcExports] [-PreserveEtcHosts] [-Controller <NaController>] 
    [<CommonParameters>]
    
    Set-NaVfilerAddress [-Name] <String> [-AddAddresses <String[]>] -Interface 
    <String> -Netmask <String> [-PreserveEtcExports] [-PreserveEtcHosts] 
    [-Controller <NaController>] [<CommonParameters>]
    
    Set-NaVfilerAddress [-Name] <String> -Addresses <String[]> -Interface 
    <String> -Netmask <String> [-PreserveEtcExports] [-PreserveEtcHosts] 
    [-Controller <NaController>] [<CommonParameters>]
    
    
DESCRIPTION
    Manage vfiler addresses.  This cmdlet can add addresses, remove addresses, 
    and set IP address bindings for a vfiler.
    

PARAMETERS
    -Name <String>
        Name of the vfiler.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -AddAddresses <String[]>
        List of IP addresses to add to this vfiler.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -RemoveAddresses <String[]>
        List of IP addresses to remove from this vfiler.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -IpBindingInfo <IpbindingInfo[]>
        List of all IP address bindings to apply.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -PreserveEtcExports
        This cmdlet uses the vfiler-setup API which will re-write the vfiler's 
        /etc/exports file.  Sometimes, this behavior is not desirable.  Use 
        this switch to automatically restore the contents of the /etc/exports 
        file once the command completes.
        
        If this parameter is not specified, the vfiler's /etc/exports file can 
        still be restored manually from the /etc/exports.bak backup.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -PreserveEtcHosts
        This cmdlet uses the vfiler-setup API which will re-write the vfiler's 
        /etc/hosts file.  Sometimes, this behavior is not desirable.  Use this 
        switch to automatically restore the contents of the /etc/hosts file 
        once the command completes.
        
        If this parameter is not specified, the vfiler's /etc/hosts file can 
        still be restored manually from the /etc/hosts.bak backup.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Interface <String>
        The interface to bind the IP addresses to.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Netmask <String>
        Netmask of the IP address binding.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Addresses <String[]>
        List of addresses to bind.
        
        Required?                    true
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216). 
    
INPUTS
    
        
        
    
     
    
OUTPUTS
    DataONTAP.Types.Vfiler.VfilerInfo
        
        
    
     
    
NOTES
    
    
        Category: vfiler
        API: vfiler-add-ipaddress, vfiler-remove-ipaddress, vfiler-setup
    
    --------------  Example 1 --------------
    
    C:\PS>Set-NaVfilerAddress vfiler2 -AddAddresses 10.10.10.22
    
    
    Add an address to vfiler 'vfiler2'.
    
    
    
    Name    Status  Ipspace         #Vfnets #Vfstores AdminHost
    ----    ------  -------         ------- --------- ---------
    vfiler2 running default-ipspace       3         1
    
    
    
    --------------  Example 2 --------------
    
    C:\PS>$b = New-Object DataONTAP.Types.Vfiler.IpbindingInfo
    $b.Interface = "e0b"
    $b.Ipaddress = "10.10.10.22"
    $b.Netmask = "255.255.255.0"
    Set-NaVfilerAddress vfiler2 -IpBindingInfo $b
    
    
    Add an IP address binding to vfiler 'vfiler2'.
    
    
    
    Name    Status  Ipspace         #Vfnets #Vfstores AdminHost
    ----    ------  -------         ------- --------- ---------
    vfiler2 running default-ipspace       3         1
    
    
    
    --------------  Example 3 --------------
    
    C:\PS>Set-NaVfilerAddress vfiler1 -AddAddresses 192.168.1.115,192.168.1.116 
    -Interface e0a -Netmask 255.255.255.0
    
    
    Add the addresses 192.168.1.116 and 192.168.1.117 to vfiler1 and bind them 
    to interface e0a with a netmask of 255.255.255.0.
    
    
    
    Name                      Status         Ipspace              VfnetCount 
    VfstoreCount AdminHost
    ----                      ------         -------              ---------- 
    ------------ ---------
    vfiler1                   running        default-ipspace               3    
            1
    
    
    
    --------------  Example 4 --------------
    
    C:\PS>Set-NaVfilerAddress vfiler1 -Addresses 192.168.1.111 -Interface e0a 
    -Netmask 255.255.255.0
    
    
    Configure the IP address 192.168.1.111 on vfiler1 binding to interface e0a 
    with netmask 255.255.255.0.
    
    
    
    Name                      Status         Ipspace              VfnetCount 
    VfstoreCount AdminHost
    ----                      ------         -------              ---------- 
    ------------ ---------
    vfiler1                   running        default-ipspace               1    
            1
