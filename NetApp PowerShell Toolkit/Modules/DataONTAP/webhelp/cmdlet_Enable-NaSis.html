Cmdlet: Enable-NaSis


NAME
    Enable-NaSis
    
SYNOPSIS
    Enables SIS on a volume.
    
SYNTAX
    Enable-NaSis [-Path] <String> [-Controller <NaController>] 
    [<CommonParameters>]
    
    Enable-NaSis [-Volume] <VolumeInfo> [-Controller <NaController>] 
    [<CommonParameters>]
    
    
DESCRIPTION
    Enables SIS on a volume. On a non-SnapVault secondary volume, the SIS 
    operation will be started periodically according to a per-volume schedule. 
    On a SnapVault secondary volume, the SIS operation will be kicked off at 
    the end of the SnapVault transfer. See the "Set-NaSis" cmdlet for the 
    details about the schedule. Users can also start SIS operation with 
    "Start-NaSis" cmdlet.
    

PARAMETERS
    -Path <String>
        The full path of the SIS volume, /vol/<vol_name>. Only one path can be 
        specified at a time. The volume must be online to enable SIS operation. 
         The /vol/ prefix may be omitted.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue, ByPropertyName)
        Accept wildcard characters?  false
        
    -Controller <NaController>
        The Data ONTAP controller object, embodied by an NaController object.  
        This parameter is returned by the Connect-NaController cmdlet.  If not 
        specified, the value in the global variable CurrentNaController is used.
        
        Required?                    false
        Position?                    named
        Default value                
        Accept pipeline input?       false
        Accept wildcard characters?  false
        
    -Volume <VolumeInfo>
        Rather than specifying the volume name, this cmdlet alternatively 
        accepts a VolumeInfo object.
        
        Required?                    true
        Position?                    1
        Default value                
        Accept pipeline input?       true (ByValue)
        Accept wildcard characters?  false
        
    <CommonParameters>
        This cmdlet supports the common parameters: Verbose, Debug,
        ErrorAction, ErrorVariable, WarningAction, WarningVariable,
        OutBuffer and OutVariable. For more information, see 
        about_CommonParameters (http://go.microsoft.com/fwlink/?LinkID=113216).
